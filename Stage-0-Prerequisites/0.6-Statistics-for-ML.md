## Key Takeaways — Read This Before Any Project

1. **Mean is sensitive to outliers — median is not.** A single $10M salary in a dataset of $50K salaries pulls the mean far from the typical value. Use median for skewed data.
2. **Z-score normalization = `(x - mean) / std`** — this is exactly what `StandardScaler` does. Understanding why: it forces every feature onto the same scale so no feature dominates due to its units.
3. **Variance tells you spread — std deviation is just variance in the original units.** `std = sqrt(variance)`. You work with std because it has the same units as your data.
4. **Normal distribution is the assumption behind linear models, z-scores, and many statistical tests.** When your data violates normality, you need to transform it or use a different model.
5. **Correlation is covariance divided by both standard deviations — scaled to [-1, 1].** Covariance tells you direction; correlation tells you both direction AND strength.
6. **Central Limit Theorem:** Even if individual data points are not normally distributed, the mean of many samples from that distribution IS approximately normal. This is why statistics still works on messy real-world data.
7. **p-value < 0.05** = the result is statistically significant (less than 5% chance it happened by random chance). This is used in A/B testing, feature importance, and model comparison.

## SECTION 1 — Descriptive Statistics

### Mean, Median, Mode — The Measures of Center ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | Three ways to describe the "typical" value in a dataset. |
| **Why** | They answer different questions about your data. Using the wrong one leads to wrong preprocessing and wrong model assumptions. |
| **When** | EDA, feature engineering, choosing imputation strategy. |
| **Where** | Every ML pipeline — before training and during evaluation. |
| **How** | Mean = sum/count. Median = middle value. Mode = most frequent. |

```python
import numpy as np
import pandas as pd
from scipy import stats
import matplotlib.pyplot as plt
import seaborn as sns

sns.set_theme(style='whitegrid', palette='muted')
np.random.seed(42)

# Normal data — mean and median are close
normal_salaries = np.random.normal(70000, 15000, 1000)

# Skewed data — mean is pulled toward outliers
skewed_salaries = np.concatenate([
    np.random.normal(60000, 10000, 980),
    np.array([500000, 800000, 1000000, 1200000, 2000000])  # 5 executives
])

print("=== Normal Data ===")
print(f"Mean:   {np.mean(normal_salaries):,.0f}")
print(f"Median: {np.median(normal_salaries):,.0f}")
print(f"Std:    {np.std(normal_salaries):,.0f}")

print("\n=== Skewed Data (5 executives added) ===")
print(f"Mean:   {np.mean(skewed_salaries):,.0f}")    # pulled high by executives
print(f"Median: {np.median(skewed_salaries):,.0f}")  # not affected by executives
print(f"Mode:   {stats.mode(skewed_salaries.astype(int))[0]:,}")

# Visualize the difference
fig, axes = plt.subplots(1, 2, figsize=(13, 4))
for ax, data, title in zip(axes,
    [normal_salaries, skewed_salaries],
    ['Normal Salary Distribution', 'Skewed Salary (with executives)']):
    ax.hist(data, bins=50, color='steelblue', edgecolor='white', alpha=0.7)
    ax.axvline(np.mean(data),   color='red',    linewidth=2, linestyle='--', label=f'Mean: {np.mean(data):,.0f}')
    ax.axvline(np.median(data), color='green',  linewidth=2, linestyle='-',  label=f'Median: {np.median(data):,.0f}')
    ax.set_title(title)
    ax.legend()
plt.tight_layout()
plt.show()
```

> **ML decision:** When imputing missing values — use **median** for skewed features (salary, price, income), **mean** for symmetric features. Using mean on skewed data distorts the imputed value toward the outliers.

### Variance and Standard Deviation ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | Variance = average of squared distances from the mean. Std = square root of variance (same units as data). |
| **Why** | Tells you how spread out your data is. High std = values far from mean. Low std = values clustered near mean. ML models behave very differently on tight vs spread features. |
| **When** | Normalization, PCA, detecting noisy features. |
| **How** | Variance: `Σ(x - mean)² / n`. Std: `sqrt(variance)`. |

```python
a = np.array([10, 10, 10, 10, 10])    # no spread
b = np.array([2, 6, 10, 14, 18])      # spread out
c = np.array([1, 2, 10, 18, 19])      # very spread

for name, data in [('No spread', a), ('Medium spread', b), ('High spread', c)]:
    print(f"{name:15s} | mean={np.mean(data):.1f} | var={np.var(data):.1f} | std={np.std(data):.1f}")

# Why squaring? Because (x - mean) can be negative — squaring makes all terms positive
# Why take sqrt? To get back to original units
# Salary std of $15,000 means most values within $15K of the mean — interpretable
```

> **Used in Stage 1:** PCA (Principal Component Analysis) finds the directions (principal components) where **variance is highest** — those directions carry the most information. Low-variance features are often dropped because they don't contribute much signal.

### Z-Score — Standardization ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | Transform a value into "how many standard deviations it is from the mean." |
| **Why** | This is the most important normalization technique in ML. It puts every feature on the same scale so features with large values (salary: 70000) don't dominate features with small values (age: 35). |
| **When** | Before training any distance-based model (KNN, SVM, neural networks, linear regression). |
| **Formula** | `z = (x - mean) / std` |

```python
# Manual z-score
data = np.array([20000, 45000, 70000, 95000, 200000])
mean = np.mean(data)
std  = np.std(data)
z_scores = (data - mean) / std

print("Original:", data)
print("Z-scores:", z_scores.round(2))
print(f"\nMean of z-scores: {z_scores.mean():.6f}")   # always 0
print(f"Std of z-scores:  {z_scores.std():.6f}")     # always 1

# This is exactly what sklearn's StandardScaler does
from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
scaled = scaler.fit_transform(data.reshape(-1, 1))
print("\nStandardScaler output:", scaled.flatten().round(2))
print("Same result:", np.allclose(z_scores, scaled.flatten()))

# Visualize before and after
fig, axes = plt.subplots(1, 2, figsize=(12, 4))
axes[0].hist(data, bins=20, color='steelblue', edgecolor='white')
axes[0].set_title(f'Before: mean={mean:.0f}, std={std:.0f}')
axes[0].set_xlabel('Value')

axes[1].hist(z_scores, bins=20, color='seagreen', edgecolor='white')
axes[1].set_title('After z-score: mean≈0, std≈1')
axes[1].set_xlabel('Z-score')
plt.tight_layout()
plt.show()
```

> **Used in Stage 1:** `StandardScaler` is applied before KNN, SVM, linear regression, and neural networks. Now you know it's just `(x - mean) / std` — not magic.

### Percentiles and IQR ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | Percentile = value below which X% of data falls. IQR (Interquartile Range) = Q3 - Q1 = range of the middle 50%. |
| **Why** | Percentiles describe spread without being affected by outliers. IQR is the foundation of the outlier detection method you used in EDA. |
| **When** | Outlier detection, `RobustScaler`, comparing model performance distributions. |

```python
data = np.random.exponential(50000, 1000)  # skewed salary data

p25 = np.percentile(data, 25)   # Q1
p50 = np.percentile(data, 50)   # Q2 = median
p75 = np.percentile(data, 75)   # Q3
iqr = p75 - p25

print(f"Q1 (25th):  {p25:,.0f}")
print(f"Q2 (50th):  {p50:,.0f}  ← median")
print(f"Q3 (75th):  {p75:,.0f}")
print(f"IQR:        {iqr:,.0f}")
print(f"Outlier boundaries: [{p25 - 1.5*iqr:,.0f},  {p75 + 1.5*iqr:,.0f}]")

# RobustScaler uses median and IQR instead of mean and std
# It is outlier-resistant — z-score is not
from sklearn.preprocessing import RobustScaler
robust = RobustScaler()
scaled_robust = robust.fit_transform(data.reshape(-1, 1))
# Internally: (x - median) / IQR
```

> **Used in Stage 1:** `RobustScaler` uses median and IQR. Use it when your data has significant outliers you cannot remove. `StandardScaler` uses mean and std — distorted by outliers. Knowing the math helps you choose.

## SECTION 2 — Probability Distributions

### Normal Distribution ⭐⭐⭐ 🧠 Shape + Properties

| | |
|---|---|
| **What** | A symmetric bell-shaped distribution defined by mean (center) and std (width). |
| **Why** | The most important distribution in statistics and ML. Linear regression assumes normally distributed errors. Z-scores are meaningful only for normal data. Many ML algorithms implicitly assume normality. |
| **When** | Understanding model assumptions, deciding when to normalize, interpreting residuals. |
| **Properties** | ~68% of data within 1 std. ~95% within 2 std. ~99.7% within 3 std. |

```python
from scipy.stats import norm

x = np.linspace(-4, 4, 300)

fig, axes = plt.subplots(1, 2, figsize=(14, 5))

# Different means and stds
for mu, sigma, color, label in [
    (0,   1,    'steelblue', 'μ=0, σ=1 (standard normal)'),
    (2,   1,    'tomato',    'μ=2, σ=1 (shifted right)'),
    (0,   2,    'seagreen',  'μ=0, σ=2 (wider)'),
]:
    axes[0].plot(x, norm.pdf(x, mu, sigma), color=color, linewidth=2, label=label)

axes[0].set_title('Normal Distributions — Effect of Mean and Std')
axes[0].set_xlabel('Value')
axes[0].set_ylabel('Probability Density')
axes[0].legend()
axes[0].grid(True, alpha=0.3)

# The 68-95-99.7 rule
x2 = np.linspace(-4, 4, 300)
y2 = norm.pdf(x2, 0, 1)
axes[1].plot(x2, y2, 'black', linewidth=2)
axes[1].fill_between(x2, y2, where=(abs(x2) <= 1), alpha=0.5, color='steelblue', label='68% (±1σ)')
axes[1].fill_between(x2, y2, where=(abs(x2) <= 2), alpha=0.3, color='orange',   label='95% (±2σ)')
axes[1].fill_between(x2, y2, where=(abs(x2) <= 3), alpha=0.2, color='green',    label='99.7% (±3σ)')
axes[1].set_title('68-95-99.7 Rule')
axes[1].set_xlabel('Standard deviations from mean')
axes[1].legend()
axes[1].grid(True, alpha=0.3)

plt.tight_layout()
plt.show()

print("Probability within ±1σ:", norm.cdf(1) - norm.cdf(-1))
print("Probability within ±2σ:", norm.cdf(2) - norm.cdf(-2))
print("Probability within ±3σ:", norm.cdf(3) - norm.cdf(-3))
```

> **Used in Stage 1:** Linear regression assumes residuals (prediction errors) are normally distributed. If you plot the residuals and they're skewed — your model violates this assumption and may produce unreliable confidence intervals.

> **Used in Stage 2:** Weights in neural networks are initialized from a normal distribution `N(0, 0.01)`. Starting with normal random weights is one of the key insights that makes deep learning work.

### Other Distributions You'll Encounter ⭐⭐ 📖

```python
from scipy import stats

x_pos = np.linspace(0, 10, 300)

fig, axes = plt.subplots(2, 2, figsize=(13, 9))

# Exponential — time between events, income, count data (always right-skewed)
axes[0,0].plot(x_pos, stats.expon.pdf(x_pos, scale=2), color='tomato', linewidth=2)

axes[0,1].set_title('Uniform\n→ random noise, dropout masks\n→ np.random.rand() generates this')
axes[0,1].set_xlabel('Value')

# Binomial — count of successes in N trials
k = np.arange(0, 21)
axes[1,0].bar(k, stats.binom.pmf(k, n=20, p=0.3), color='seagreen', edgecolor='white')
axes[1,0].set_title('Binomial (n=20, p=0.3)\n→ click-through rates, pass/fail\n→ basis of logistic regression')
axes[1,0].set_xlabel('Number of successes')

# Poisson — count of rare events per interval
k2 = np.arange(0, 15)
axes[1,1].bar(k2, stats.poisson.pmf(k2, mu=3), color='gold', edgecolor='white')
axes[1,1].set_title('Poisson (λ=3)\n→ transactions per hour, defects per unit\n→ used in fraud detection')
axes[1,1].set_xlabel('Count')

plt.suptitle('Common Distributions in ML', fontsize=13)
plt.tight_layout()
plt.show()
```

> **Why distributions matter for model choice:**
> - Continuous target → Linear Regression (assumes normal errors)
> - Binary target (0/1) → Logistic Regression (Binomial distribution)
> - Count target → Poisson Regression
> - Heavy-tailed data → Tree-based models (don't assume any distribution)

## SECTION 3 — Covariance and Correlation

### Covariance ⭐⭐⭐ 🧠 Concept, 📖 Code

| | |
|---|---|
| **What** | Measures how two variables change together. Positive = both go up together. Negative = one goes up when the other goes down. |
| **Why** | The raw building block of correlation, PCA, and many ML algorithms. |
| **When** | Understanding feature relationships, building the correlation matrix. |
| **Problem** | Covariance values depend on the units of the features — you can't compare covariates across different feature pairs. |

```python
np.random.seed(42)
n = 200

age        = np.random.randint(22, 60, n)
salary     = age * 2000 + np.random.randn(n) * 10000   # correlated with age
score      = 100 - age * 0.3 + np.random.randn(n) * 10  # negatively correlated

# Covariance matrix
X = np.column_stack([age, salary, score])
cov_matrix = np.cov(X.T)   # .T because np.cov expects each variable as a row

print("Covariance Matrix:")
print(pd.DataFrame(cov_matrix,
    index=['age','salary','score'],
    columns=['age','salary','score']).round(0))

print("\nInterpretation:")
print(f"age-salary covariance:  {cov_matrix[0,1]:,.0f}  (positive → both go up together)")
print(f"age-score covariance:   {cov_matrix[0,2]:,.2f}  (negative → as age rises, score falls)")
print(f"age-age covariance:     {cov_matrix[0,0]:,.0f}  (this is just variance of age)")
```

> **Problem with covariance:** `age-salary` covariance might be `500,000` while `age-score` covariance is `-3`. Are they equally strong relationships? You can't tell because the scales are different. This is why we use **correlation** — it normalizes covariance to [-1, 1].

---

### Correlation — Normalized Covariance ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | Covariance divided by the product of both standard deviations — scales to [-1, 1]. |
| **Why** | Makes relationships comparable across all feature pairs regardless of units. -1 = perfect negative, 0 = no relationship, +1 = perfect positive. |
| **When** | Feature selection, multicollinearity detection, understanding the dataset. |
| **Formula** | `r = cov(X,Y) / (std(X) × std(Y))` |

```python
# Pearson correlation (linear relationship)
corr_matrix = np.corrcoef(X.T)
print("Correlation Matrix:")
print(pd.DataFrame(corr_matrix,
    index=['age','salary','score'],
    columns=['age','salary','score']).round(3))

# Visualize
df_temp = pd.DataFrame({'age': age, 'salary': salary, 'score': score})
fig, axes = plt.subplots(1, 3, figsize=(15, 4))

for ax, (col_x, col_y), color in zip(axes,
    [('age','salary'), ('age','score'), ('salary','score')],
    ['steelblue', 'tomato', 'seagreen']):
    r = df_temp[col_x].corr(df_temp[col_y])
    ax.scatter(df_temp[col_x], df_temp[col_y], alpha=0.4, color=color, s=20)
    ax.set_xlabel(col_x)
    ax.set_ylabel(col_y)
    ax.set_title(f'{col_x} vs {col_y}\nr = {r:.2f}')

plt.suptitle('Correlation Examples', fontsize=13)
plt.tight_layout()
plt.show()

# Important: correlation only measures LINEAR relationships
# Two features can have strong non-linear relationship but r ≈ 0
x_circ = np.linspace(-1, 1, 200)
y_circ = x_circ**2 + np.random.randn(200) * 0.05
print(f"\nQuadratic relationship (y=x²) — Pearson r: {np.corrcoef(x_circ, y_circ)[0,1]:.3f}")
print("Near 0 even though there's a clear pattern — Pearson misses non-linear relationships")
```

> **Used in Stage 1:** Before feature selection, run `df.corr()`. Features with `|r| > 0.85` with each other are redundant — drop one. Features with `|r| > 0.3` with the target are worth keeping. This is quantitative feature selection.

## SECTION 4 — Central Limit Theorem

### CLT — Why Statistics Works on Messy Data ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | The Central Limit Theorem states: if you take the mean of a large enough sample from ANY distribution, that sample mean will be approximately normally distributed — regardless of what the original distribution looks like. |
| **Why** | This is one of the most important results in all of statistics. It explains why we can use statistical tests that assume normality on real-world data that isn't normal. It also explains why averaging many predictions works in ensemble models. |
| **When** | Understanding why ML evaluation metrics are reliable. Why averaging results over k-fold cross-validation gives you a stable estimate. |
| **How much** | Understand the intuition deeply — this will come back in model evaluation and ensemble methods. |

```python
np.random.seed(42)

# Original distribution — very skewed (exponential)
population = np.random.exponential(scale=10, size=100000)

# Take many samples and compute their means
sample_sizes = [5, 30, 100]
n_samples    = 2000

fig, axes = plt.subplots(2, 2, figsize=(14, 9))

# Original distribution
axes[0,0].hist(population, bins=80, color='tomato', edgecolor='white', alpha=0.8, density=True)
axes[0,0].set_title(f'Original Population\n(Exponential — heavily skewed, skew={pd.Series(population).skew():.2f})')
axes[0,0].set_xlabel('Value')
axes[0,0].set_ylabel('Density')

# Sample means for increasing sample sizes
for ax, n in zip(axes.flatten()[1:], sample_sizes):
    sample_means = [np.mean(np.random.choice(population, size=n)) for _ in range(n_samples)]
    ax.hist(sample_means, bins=50, color='steelblue', edgecolor='white', alpha=0.8, density=True)
    # Overlay theoretical normal
    mu  = np.mean(sample_means)
    sig = np.std(sample_means)
    x_range = np.linspace(mu - 4*sig, mu + 4*sig, 200)
    ax.plot(x_range, norm.pdf(x_range, mu, sig), 'r-', linewidth=2, label='Normal fit')
    ax.set_title(f'Sample Means (n={n})\nskew={pd.Series(sample_means).skew():.2f}  → approaches normal')
    ax.set_xlabel('Sample Mean')
    ax.set_ylabel('Density')
    ax.legend()

plt.suptitle('Central Limit Theorem — Skewed Data Becomes Normal as Sample Size Grows',
             fontsize=12)
plt.tight_layout()
plt.show()

print("KEY INSIGHT:")
print("Original data skewness:     ", round(pd.Series(population).skew(), 2))
for n in sample_sizes:
    means = [np.mean(np.random.choice(population, size=n)) for _ in range(2000)]
    print(f"Sample means (n={n:3d}) skew: {round(pd.Series(means).skew(), 3)}")
print("\nConclusion: as n grows, distribution of means approaches normal.")
```

> **Used in Stage 1:** When you run k-fold cross-validation (say 10 folds), the 10 accuracy scores you get are samples from the "true model accuracy distribution." By CLT, their mean is approximately normally distributed — so you can compute a confidence interval around your model's accuracy. This is scientifically sound even if the underlying errors aren't normal.

> **Used in Stage 2:** Batch normalization in neural networks works partly because of CLT — normalizing activations across a batch stabilizes the distribution of inputs to each layer.

## SECTION 5 — Hypothesis Testing

### p-value and Statistical Significance ⭐⭐⭐ 🧠 Concept, 📖 Code

| | |
|---|---|
| **What** | A hypothesis test asks: "Is the difference/pattern I'm seeing real, or could it have happened by random chance?" The p-value is the probability that the result occurred by chance if there's actually no real effect. |
| **Why** | In ML you constantly compare things: is Model A actually better than Model B? Is Feature X actually predictive or just correlated by chance? p-values give you a principled answer. |
| **When** | A/B testing models in production. Feature importance testing. Comparing model performance across experiments. |
| **Rule** | p-value < 0.05 → result is statistically significant (less than 5% chance it's random) |

```python
from scipy import stats

np.random.seed(42)

# Scenario: Did the new feature improve model accuracy?
# Model A (baseline): 10 accuracy scores from cross-validation
# Model B (new feature): 10 accuracy scores from cross-validation
model_a = np.array([0.81, 0.79, 0.83, 0.80, 0.82, 0.78, 0.81, 0.80, 0.83, 0.79])
model_b = np.array([0.85, 0.86, 0.84, 0.87, 0.85, 0.86, 0.85, 0.87, 0.84, 0.86])

print(f"Model A mean accuracy: {model_a.mean():.3f} ± {model_a.std():.3f}")
print(f"Model B mean accuracy: {model_b.mean():.3f} ± {model_b.std():.3f}")
print(f"Difference:            {model_b.mean() - model_a.mean():.3f}")

# Paired t-test — are they significantly different?
t_stat, p_value = stats.ttest_rel(model_a, model_b)
print(f"\nt-statistic: {t_stat:.3f}")
print(f"p-value:     {p_value:.6f}")

if p_value < 0.05:
    print("✓ SIGNIFICANT — Model B is genuinely better (p < 0.05)")
else:
    print("✗ NOT SIGNIFICANT — difference could be random chance")

# Visualize
fig, ax = plt.subplots(figsize=(8, 5))
ax.boxplot([model_a, model_b], labels=['Model A\n(baseline)', 'Model B\n(new feature)'],
           patch_artist=True,
           boxprops=dict(facecolor='steelblue', alpha=0.7))
ax.set_title(f'Model Comparison\np-value={p_value:.4f} → {"Significant" if p_value < 0.05 else "Not significant"}')
ax.set_ylabel('Accuracy')
ax.grid(True, alpha=0.3)
plt.tight_layout()
plt.show()
```

> **Used in Stage 7:** In production, when you deploy a new model version and run an A/B test, you use a t-test (or similar) to confirm the improvement is real before fully rolling out. Without this, you might be chasing random noise.

### Confidence Intervals ⭐⭐ 🧠 Concept, 📖 Code

| | |
|---|---|
| **What** | A range of values that, with a specified confidence (usually 95%), contains the true population parameter. |
| **Why** | A single accuracy number (e.g., "my model is 85% accurate") is incomplete. A confidence interval says "85% ± 2%" — telling you how reliable that estimate is. |
| **When** | Reporting model performance, comparing models, communicating results to stakeholders. |

```python
from scipy.stats import t as t_dist

def confidence_interval(data, confidence=0.95):
    n    = len(data)
    mean = np.mean(data)
    se   = stats.sem(data)          # standard error = std / sqrt(n)
    ci   = se * t_dist.ppf((1 + confidence) / 2, df=n - 1)
    return mean, mean - ci, mean + ci

mean, lower, upper = confidence_interval(model_b)
print(f"Model B accuracy: {mean:.3f}")
print(f"95% CI: [{lower:.3f}, {upper:.3f}]")
print(f"Interpretation: we are 95% confident the true accuracy is between {lower:.1%} and {upper:.1%}")

# Bootstrap CI — more flexible, doesn't assume normality
n_bootstrap = 10000
bootstrap_means = [np.mean(np.random.choice(model_b, size=len(model_b), replace=True))
                   for _ in range(n_bootstrap)]
boot_lower = np.percentile(bootstrap_means, 2.5)
boot_upper = np.percentile(bootstrap_means, 97.5)
print(f"\nBootstrap 95% CI: [{boot_lower:.3f}, {boot_upper:.3f}]")
```

> **Used in Stage 7:** When you report model performance to a business stakeholder, never say just "85% accuracy." Say "85% accuracy with 95% CI [83%, 87%]." That range tells them how reliable the number is.

### Choosing the Right Scaler Based on Distribution ⭐⭐⭐ 🧠

```python
from sklearn.preprocessing import StandardScaler, MinMaxScaler, RobustScaler
import numpy as np

data = np.array([10, 12, 11, 13, 10, 100, 11, 12, 10, 11])  # 100 is an outlier

s1 = StandardScaler().fit_transform(data.reshape(-1,1)).flatten()
s2 = MinMaxScaler().fit_transform(data.reshape(-1,1)).flatten()
s3 = RobustScaler().fit_transform(data.reshape(-1,1)).flatten()

print("Original:        ", data)
print("StandardScaler:  ", s1.round(2), "← outlier pulls mean, distorts others")
print("MinMaxScaler:    ", s2.round(2), "← outlier compresses all values to near 0")
print("RobustScaler:    ", s3.round(2), "← outlier barely affects the scaling")
```

| Scaler | Uses | When to use |
|--------|------|------------|
| `StandardScaler` | mean, std | Data is roughly normal, no major outliers |
| `MinMaxScaler` | min, max | When you need values in [0, 1], no outliers |
| `RobustScaler` | median, IQR | Data has significant outliers you can't remove |

### Statistical Summary of a Feature — Full Picture ⭐⭐⭐ 🧠

```python
def full_stats(series, name='feature'):
    s = series.dropna()
    Q1, Q3 = np.percentile(s, [25, 75])
    IQR    = Q3 - Q1
    print(f"\n{'='*40}")
    print(f"  {name}")
    print(f"{'='*40}")
    print(f"  Count:      {len(s):,}")
    print(f"  Mean:       {s.mean():,.2f}")
    print(f"  Median:     {np.median(s):,.2f}")
    print(f"  Std:        {s.std():,.2f}")
    print(f"  Min/Max:    {s.min():,.2f} / {s.max():,.2f}")
    print(f"  Q1/Q3:      {Q1:,.2f} / {Q3:,.2f}")
    print(f"  IQR:        {IQR:,.2f}")
    print(f"  Skewness:   {s.skew():.3f}", end='  ')
    if   abs(s.skew()) < 0.5: print("(approx normal)")
    elif abs(s.skew()) < 1.0: print("(moderate skew)")
    else:                     print("⚠ HIGH SKEW — consider log transform")
    print(f"  Outliers:   {((s < Q1-1.5*IQR) | (s > Q3+1.5*IQR)).sum()} "
          f"({((s < Q1-1.5*IQR) | (s > Q3+1.5*IQR)).mean()*100:.1f}%)")

})
```

## Summary — The Mental Model

```
Describing a single feature:
  Center    →  mean (normal data) or median (skewed data)
  Spread    →  std (or IQR for outlier-resistant)
  Shape     →  skewness → decide whether to log-transform
  Outliers  →  IQR rule → Q1-1.5×IQR, Q3+1.5×IQR

Normalization:
  z-score   →  (x - mean) / std  →  StandardScaler
  robust    →  (x - median) / IQR → RobustScaler
  min-max   →  (x - min) / (max - min) → MinMaxScaler

Feature relationships:
  Covariance  →  direction only (units matter)
  Correlation →  direction + strength, scale-free [-1, 1]

Why things work:
  CLT         →  averages become normal → statistics is reliable
  p-value     →  is this real or random noise?
  CI          →  how confident are we in this number?
```
