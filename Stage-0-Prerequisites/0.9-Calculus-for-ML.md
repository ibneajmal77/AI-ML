## Where This Fits in the AI/ML Pipeline

```
[ LINEAR ALGEBRA ]
  Data and weights are matrices
          ↓
[ CALCULUS ]            ← YOU ARE HERE
  The engine of learning.
  "How do I change my weights
   to make my predictions better?"
  Calculus answers this by computing:
  - How much does the loss change
    when I nudge each weight?
  - In which direction should I
    move each weight to reduce loss?
          ↓
[ GRADIENT DESCENT ]
  Use those derivatives to
  update weights step by step
          ↓
[ TRAINED MODEL ]
  Weights that minimize loss
  on training data
```

> Statistics describes your data. Linear algebra holds your model. Calculus **trains** your model. Without derivatives, gradient descent does not exist. Without gradient descent, neural networks cannot learn. Every `optimizer.step()` you call in PyTorch is calculus running under the hood.

## Key Takeaways — Read This Before Any Project

1. **A derivative tells you the slope — the gradient tells you the direction to climb.** You want to *descend*, so you move in the *negative* gradient direction. This is gradient descent.
2. **The chain rule IS backpropagation.** When computing how the loss changes with respect to the first layer's weights, you chain together derivatives through every layer. PyTorch's autograd does this automatically, but understanding it means you can debug NaN losses and understand why deep networks are hard to train.
3. **Learning rate is the most important hyperparameter in training.** Too high → loss oscillates or diverges. Too low → training takes forever or gets stuck. Adam optimizer partially solves this by adapting the learning rate per parameter.
4. **Vanishing gradients killed deep learning until ReLU.** Sigmoid's derivative is at most 0.25 — multiply it across 10 layers and the gradient shrinks to essentially zero. ReLU's derivative is 1 for positive inputs — gradients flow freely.
5. **Loss landscapes in deep learning are not convex.** There are local minima, saddle points, and flat regions. Yet gradient descent works well in practice because in high dimensions, most critical points are saddle points, not bad local minima.
6. **You never manually compute derivatives in ML.** PyTorch/TensorFlow's autograd does it. But you need to understand what `loss.backward()` is computing — otherwise you cannot debug training failures, NaN losses, or gradient issues.

## CONCEPT 1 — Derivatives: The Slope at a Point

| Question | Answer |
|----------|--------|
| **What** | The derivative `dy/dx` is the instantaneous rate of change of y with respect to x |
| **Why** | It tells you: "if I increase x by a tiny amount, how much does y change?" — exactly what you need to know which way to adjust weights |
| **When** | Every training step — you compute the derivative of the loss with respect to every weight |
| **Where** | Foundation of every optimizer: SGD, Adam, RMSprop |
| **Who** | Every ML engineer needs the conceptual understanding; PyTorch handles computation |
| **How** | Geometrically: slope of the tangent line at a point on the curve |
| **How often** | Every training iteration, millions of times per training run |
| **How much** | Memorize the concept and intuition. Know basic rules. Rely on autograd for actual computation. |

```python
import numpy as np
import matplotlib.pyplot as plt

# ── Numerical derivative — approximate the slope ───────────────────────────────
# The definition: f'(x) = lim(h->0) [f(x+h) - f(x)] / h

def numerical_derivative(f, x, h=1e-5):
    return (f(x + h) - f(x - h)) / (2 * h)   # central difference — more accurate

# Loss function example: L(w) = w^2 (simplest convex bowl)
f = lambda w: w**2

# At w=3: true derivative = 2*3 = 6
w = 3.0
deriv = numerical_derivative(f, w)
print(f'f(w) = w^2 at w={w}:')
print(f'  Numerical derivative: {deriv:.6f}')
print(f'  True derivative 2w:   {2*w:.6f}')

# ── Visualize — derivative = slope of tangent line ─────────────────────────────
w_vals  = np.linspace(-3, 3, 300)
loss    = w_vals**2
deriv_vals = 2 * w_vals   # d/dw (w^2) = 2w

fig, axes = plt.subplots(1, 2, figsize=(13, 5))

axes[0].plot(w_vals, loss, 'steelblue', linewidth=2, label='L(w) = w²')
# Tangent line at w=2
w0 = 2.0
slope = 2 * w0
tangent = slope * (w_vals - w0) + w0**2
axes[0].plot(w_vals, tangent, 'tomato', linestyle='--', label=f'Tangent at w=2 (slope={slope})')
axes[0].scatter([w0], [w0**2], color='tomato', s=100, zorder=5)
axes[0].set_title('L(w) = w²: Derivative = Slope of Tangent')
axes[0].set_xlabel('w (weight)'); axes[0].set_ylabel('Loss')
axes[0].legend(); axes[0].set_ylim(-1, 10)

axes[1].plot(w_vals, deriv_vals, 'seagreen', linewidth=2)
axes[1].axhline(0, color='k', linestyle='--', alpha=0.5)
axes[1].axvline(0, color='k', linestyle='--', alpha=0.5, label='w=0: gradient=0 (minimum)')
axes[1].set_title("dL/dw = 2w\n(positive -> move left, negative -> move right)")
axes[1].set_xlabel('w'); axes[1].set_ylabel('dL/dw')
axes[1].legend()

plt.tight_layout()
plt.show()

# ── Key insight: derivative sign tells you direction ───────────────────────────
print('\nDerivative sign tells you where to move:')
for w_test in [-2.0, -0.5, 0.0, 1.0, 2.5]:
    d = 2 * w_test
    direction = 'move LEFT (decrease w)' if d > 0 else ('move RIGHT (increase w)' if d < 0 else 'AT MINIMUM')
    print(f'  w={w_test:5.1f}  dL/dw={d:6.1f}  -> {direction}')
```

> **Real-world scenario — Neural network weight update:**
> You have a weight `w = 2.5` and the gradient is `dL/dw = 0.8`.
> The gradient is positive, meaning increasing `w` increases the loss.
> So you decrease `w`: `w = w - lr * gradient = 2.5 - 0.01 * 0.8 = 2.492`
> Do this for every weight in the network, every iteration.
> That is gradient descent — and it is calculus running billions of times per training run.

> **Used in:** Stage 1 — Every optimizer; Stage 2 — Backpropagation

## CONCEPT 2 — Partial Derivatives and Gradients: Multi-Dimensional Slopes

| Question | Answer |
|----------|--------|
| **What** | Partial derivative `∂L/∂w_i`: how much does loss change when only weight `w_i` changes, all else fixed |
| **Why** | A model has millions of weights — each needs its own gradient to know how to update |
| **When** | Every training step — one gradient per parameter |
| **Where** | Core of every neural network backward pass |
| **Who** | Every ML/DL engineer |
| **How** | Treat all other variables as constants, differentiate with respect to the one you care about |
| **How often** | Every training step, for every parameter |
| **How much** | Understand the concept. In practice, PyTorch computes all partial derivatives automatically. |

```python
import numpy as np
import matplotlib.pyplot as plt
from mpl_toolkits.mplot3d import Axes3D

# ── 2-parameter loss function ──────────────────────────────────────────────────
# L(w1, w2) = w1^2 + 2*w2^2
# Partial derivatives:
#   dL/dw1 = 2*w1    (treat w2 as constant)
#   dL/dw2 = 4*w2    (treat w1 as constant)

def loss_fn(w1, w2):
    return w1**2 + 2*w2**2

# At point (w1=2, w2=1)
w1, w2 = 2.0, 1.0
dL_dw1 = 2 * w1    # 4.0
dL_dw2 = 4 * w2    # 4.0
gradient = np.array([dL_dw1, dL_dw2])

print(f'At (w1={w1}, w2={w2}):')
print(f'  L(w1,w2)  = {loss_fn(w1,w2):.2f}')
print(f'  dL/dw1    = {dL_dw1:.2f}  (slope in w1 direction)')
print(f'  dL/dw2    = {dL_dw2:.2f}  (slope in w2 direction)')
print(f'  gradient  = {gradient}   (direction of steepest INCREASE)')
print(f'  -gradient = {-gradient}  (direction of steepest DECREASE = where to move)')

# ── Gradient descent on 2D loss surface ───────────────────────────────────────
lr = 0.1
w = np.array([2.0, 1.5])   # starting point
trajectory = [w.copy()]

for step in range(20):
    grad = np.array([2*w[0], 4*w[1]])   # gradient at current point
    w = w - lr * grad                    # gradient descent step
    trajectory.append(w.copy())

trajectory = np.array(trajectory)

print(f'\nGradient descent trajectory:')
for i in [0, 5, 10, 15, 19]:
    print(f'  step {i:2d}: w={trajectory[i]}  L={loss_fn(*trajectory[i]):.6f}')

# Visualize 3D loss surface with trajectory
w1_grid = np.linspace(-2.5, 2.5, 80)
w2_grid = np.linspace(-2.0, 2.0, 80)
W1, W2  = np.meshgrid(w1_grid, w2_grid)
L       = loss_fn(W1, W2)

fig, axes = plt.subplots(1, 2, figsize=(14, 5))

# 3D surface
ax3d = fig.add_subplot(121, projection='3d')
ax3d.plot_surface(W1, W2, L, cmap='viridis', alpha=0.6)
ax3d.plot(trajectory[:,0], trajectory[:,1],
          [loss_fn(*p) for p in trajectory],
          'r.-', linewidth=2, markersize=5, label='GD path')
ax3d.set_title('Loss Surface L(w1,w2) = w1² + 2w2²')
ax3d.set_xlabel('w1'); ax3d.set_ylabel('w2'); ax3d.set_zlabel('Loss')

# 2D contour
fig.delaxes(axes[0])   # replace with 3d
ax2d = fig.add_subplot(122)
cp = ax2d.contourf(W1, W2, L, levels=30, cmap='viridis')
ax2d.plot(trajectory[:,0], trajectory[:,1], 'r.-', linewidth=2, markersize=8)
ax2d.scatter(*trajectory[0], color='red',   s=100, zorder=5, label='start')
ax2d.scatter(*trajectory[-1], color='lime', s=100, zorder=5, label='end (minimum)')
ax2d.set_title('Gradient Descent — Contour View\n(arrows point toward minimum)')
ax2d.set_xlabel('w1'); ax2d.set_ylabel('w2')
ax2d.legend()
plt.colorbar(cp, ax=ax2d, label='Loss')
plt.tight_layout()
plt.show()
```

> **Real-world scenario — Training GPT:**
> GPT-3 has 175 billion parameters. Each parameter has its own partial derivative: `∂L/∂w_i`.
> After one forward pass on a batch of text, backpropagation computes all 175B gradients simultaneously.
> Each weight is then nudged by its gradient. This happens thousands of times per hour on hundreds of GPUs.
> The entire process — training the world's most powerful language models — is billions of partial derivatives being computed and applied.

> **Used in:** Stage 2 — Every `loss.backward()` call computes all partial derivatives

## CONCEPT 3 — The Chain Rule: The Heart of Backpropagation

| Question | Answer |
|----------|--------|
| **What** | If `y = f(g(x))`, then `dy/dx = f'(g(x)) · g'(x)` — derivative of outer times derivative of inner |
| **Why** | Neural networks are compositions of functions. The chain rule lets you compute how the loss depends on any weight deep in the network. |
| **When** | Every single backpropagation pass in every neural network |
| **Where** | The mathematical core of `loss.backward()` in PyTorch |
| **Who** | Every DL engineer must understand this to debug training failures |
| **How** | Work backwards from the loss, multiplying derivatives layer by layer |
| **How often** | Every training iteration |
| **How much** | Memorize the rule and understand its application in a 2-layer network. PyTorch does the rest. |

```python
import numpy as np
import matplotlib.pyplot as plt

# ── Chain rule example ─────────────────────────────────────────────────────────
# A simple 2-layer network with one neuron each:
# x -> [w1] -> z1 = w1*x -> relu -> a1 -> [w2] -> z2 = w2*a1 -> loss = z2^2
#
# We want: dL/dw1  (how does loss change if we tweak w1?)
# Chain rule: dL/dw1 = dL/dz2 * dz2/da1 * da1/dz1 * dz1/dw1

x  = 2.0
w1 = 0.5
w2 = 0.3

# Forward pass
z1 = w1 * x         # 1.0
a1 = max(0, z1)     # relu: 1.0  (z1 > 0, so relu = z1)
z2 = w2 * a1        # 0.3
L  = z2**2          # 0.09

print('Forward pass:')
print(f'  x={x}, w1={w1}, w2={w2}')
print(f'  z1 = w1*x   = {z1}')
print(f'  a1 = relu(z1) = {a1}')
print(f'  z2 = w2*a1  = {z2}')
print(f'  L  = z2^2   = {L}')

# Backward pass — chain rule layer by layer
dL_dz2 = 2 * z2               # d(z2^2)/dz2 = 2*z2  = 0.6
dz2_da1 = w2                  # d(w2*a1)/da1 = w2   = 0.3
da1_dz1 = 1.0 if z1 > 0 else 0.0   # relu derivative   = 1.0
dz1_dw1 = x                   # d(w1*x)/dw1 = x     = 2.0

# Chain rule: multiply them all together
dL_dw1 = dL_dz2 * dz2_da1 * da1_dz1 * dz1_dw1
dL_dw2 = dL_dz2 * a1                         # dz2/dw2 = a1

print('\nBackward pass (chain rule):')
print(f'  dL/dz2 = 2*z2            = {dL_dz2}')
print(f'  dz2/da1 = w2             = {dz2_da1}')
print(f'  da1/dz1 = relu\'(z1)     = {da1_dz1}')
print(f'  dz1/dw1 = x              = {dz1_dw1}')
print(f'  ──────────────────────────────────────')
print(f'  dL/dw1 = {dL_dz2} * {dz2_da1} * {da1_dz1} * {dz1_dw1} = {dL_dw1}')
print(f'  dL/dw2 = {dL_dz2} * {a1}                          = {dL_dw2}')

# Verify with numerical gradient
h = 1e-5
def full_forward(w1_val, w2_val):
    z1_v = w1_val * x
    a1_v = max(0, z1_v)
    z2_v = w2_val * a1_v
    return z2_v**2

dL_dw1_num = (full_forward(w1+h, w2) - full_forward(w1-h, w2)) / (2*h)
dL_dw2_num = (full_forward(w1, w2+h) - full_forward(w1, w2-h)) / (2*h)
print(f'\nNumerical verification:')
print(f'  dL/dw1 analytical={dL_dw1:.6f}  numerical={dL_dw1_num:.6f}  match={np.isclose(dL_dw1, dL_dw1_num)}')
print(f'  dL/dw2 analytical={dL_dw2:.6f}  numerical={dL_dw2_num:.6f}  match={np.isclose(dL_dw2, dL_dw2_num)}')

# Weight update
lr = 0.1
w1_new = w1 - lr * dL_dw1
w2_new = w2 - lr * dL_dw2
print(f'\nWeight update (lr={lr}):')
print(f'  w1: {w1} -> {w1_new:.4f}  (moved by -{lr}*{dL_dw1}={-lr*dL_dw1:.4f})')
print(f'  w2: {w2} -> {w2_new:.4f}  (moved by -{lr}*{dL_dw2}={-lr*dL_dw2:.4f})')

new_loss = full_forward(w1_new, w2_new)
print(f'\nLoss: {L:.6f} -> {new_loss:.6f}  (decreased? {new_loss < L})')
```

> **Real-world scenario — PyTorch training loop:**
> ```python
> for batch in dataloader:
>     optimizer.zero_grad()   # reset gradients
>     output = model(batch)   # forward pass: matrix multiplies
>     loss = criterion(output, labels)
>     loss.backward()         # backward pass: chain rule for all weights
>     optimizer.step()        # update: w = w - lr * gradient
> ```
> That single line `loss.backward()` runs the chain rule through every layer — through millions of weights — in milliseconds. PyTorch builds a computational graph during the forward pass and traverses it in reverse during backward.

> **Used in:** Stage 2 — Core of every neural network training loop

## CONCEPT 4 — Gradient Descent: Moving Downhill

| Question | Answer |
|----------|--------|
| **What** | An iterative algorithm that repeatedly moves weights in the direction that decreases the loss |
| **Why** | Finding the exact minimum of a complex loss function analytically is impossible — gradient descent finds it numerically |
| **When** | Training any ML/DL model |
| **Where** | The outer loop of every `model.fit()` or custom training loop |
| **Who** | Every ML engineer must understand this to tune training |
| **How** | `w = w - learning_rate * gradient` repeated until convergence |
| **How often** | Every training run — thousands to millions of iterations |
| **How much** | Memorize the update rule and variants (SGD, Mini-batch, Adam). Understand learning rate sensitivity. |

```python
import numpy as np
import matplotlib.pyplot as plt

# ── Gradient Descent variants on a real loss ───────────────────────────────────
np.random.seed(42)
n = 200
X_data = np.random.randn(n, 1)
y_data = 3 * X_data.flatten() + 1.5 + np.random.randn(n) * 0.5

# Loss: Mean Squared Error  L(w,b) = mean((y - (wx+b))^2)
def loss_mse(w, b):
    y_pred = w * X_data.flatten() + b
    return np.mean((y_data - y_pred)**2)

def gradients(w, b):
    y_pred = w * X_data.flatten() + b
    error  = y_pred - y_data
    dw     = 2 * np.mean(error * X_data.flatten())
    db     = 2 * np.mean(error)
    return dw, db

# ── Batch Gradient Descent (uses all data per step) ───────────────────────────
w, b   = 0.0, 0.0
lr     = 0.1
losses_bgd = []

for step in range(100):
    losses_bgd.append(loss_mse(w, b))
    dw, db = gradients(w, b)
    w -= lr * dw
    b -= lr * db

print(f'Batch GD result: w={w:.4f} (true=3.0)  b={b:.4f} (true=1.5)')

# ── Stochastic Gradient Descent (one sample per step) ─────────────────────────
w_sgd, b_sgd   = 0.0, 0.0
lr_sgd  = 0.01
losses_sgd = []
idx_order  = np.arange(n)

for epoch in range(10):
    np.random.shuffle(idx_order)
    for i in idx_order:
        xi, yi  = X_data[i, 0], y_data[i]
        y_pred  = w_sgd * xi + b_sgd
        error   = y_pred - yi
        dw      = 2 * error * xi
        db      = 2 * error
        w_sgd  -= lr_sgd * dw
        b_sgd  -= lr_sgd * db
    losses_sgd.append(loss_mse(w_sgd, b_sgd))

print(f'SGD result:      w={w_sgd:.4f} (true=3.0)  b={b_sgd:.4f} (true=1.5)')

# ── Mini-batch SGD (industry standard) ────────────────────────────────────────
w_mb, b_mb     = 0.0, 0.0
lr_mb   = 0.05
batch_size = 32
losses_mb  = []

for epoch in range(10):
    perm = np.random.permutation(n)
    for start in range(0, n, batch_size):
        idx   = perm[start:start+batch_size]
        X_b   = X_data[idx, 0]
        y_b   = y_data[idx]
        y_p   = w_mb * X_b + b_mb
        err   = y_p - y_b
        w_mb -= lr_mb * 2 * np.mean(err * X_b)
        b_mb -= lr_mb * 2 * np.mean(err)
    losses_mb.append(loss_mse(w_mb, b_mb))

print(f'Mini-batch SGD:  w={w_mb:.4f} (true=3.0)  b={b_mb:.4f} (true=1.5)')

# Plot convergence comparison
fig, axes = plt.subplots(1, 2, figsize=(14, 5))

axes[0].plot(losses_bgd, label='Batch GD',       color='steelblue', linewidth=2)
axes[0].plot(losses_sgd, label='SGD (per epoch)', color='tomato',    linewidth=2)
axes[0].plot(losses_mb,  label='Mini-batch SGD',  color='seagreen',  linewidth=2)
axes[0].set_title('Convergence: Batch vs SGD vs Mini-batch')
axes[0].set_xlabel('Epoch'); axes[0].set_ylabel('Loss')
axes[0].legend()

# Learning rate sensitivity
lrs = [0.001, 0.05, 0.1, 0.5]
colors = ['steelblue', 'seagreen', 'orange', 'tomato']
for lr_test, c in zip(lrs, colors):
    w_t, b_t = 0.0, 0.0
    traj = []
    for _ in range(50):
        traj.append(loss_mse(w_t, b_t))
        dw_t, db_t = gradients(w_t, b_t)
        w_t -= lr_test * dw_t
        b_t -= lr_test * db_t
    axes[1].plot(traj, label=f'lr={lr_test}', color=c, linewidth=2)

axes[1].set_title('Learning Rate Sensitivity')
axes[1].set_xlabel('Step'); axes[1].set_ylabel('Loss')
axes[1].legend(); axes[1].set_ylim(0, 20)

plt.tight_layout()
plt.show()
```

> **Real-world scenario — Why mini-batch SGD is used in practice:**
> ImageNet has 1.2M training images. Batch GD would compute the gradient over all 1.2M images before taking one step — computationally unaffordable and slow to converge.
> Mini-batch SGD uses 256 images per step, takes ~4,700 steps per epoch, and the noise in the gradient actually helps escape local minima.
> Every major model (ResNet, BERT, GPT) was trained with mini-batch SGD or Adam (which builds on it).

> **Used in:** Stage 1 — `sklearn` models use variants of this; Stage 2 — Core of PyTorch training

## CONCEPT 5 — Activation Functions and Their Derivatives

| Question | Answer |
|----------|--------|
| **What** | Non-linear functions applied after each layer; their derivatives determine how gradients flow backward |
| **Why** | Without non-linearity, stacking linear layers is still just one linear layer. Derivatives matter because they control whether gradients vanish. |
| **When** | Every neural network layer — the choice of activation is an architectural decision |
| **Where** | Stage 2 — Every neural network; choosing ReLU vs sigmoid vs tanh vs GELU |
| **Who** | Every DL engineer |
| **How** | Forward: apply function to layer output. Backward: multiply by its derivative. |
| **How often** | Every network you build |
| **How much** | Know sigmoid, tanh, ReLU, Leaky ReLU, GELU. Understand why ReLU is the default. |

```python
import numpy as np
import matplotlib.pyplot as plt

z = np.linspace(-4, 4, 300)

# ── Activation functions ───────────────────────────────────────────────────────
def sigmoid(z):    return 1 / (1 + np.exp(-z))
def sigmoid_d(z):  s = sigmoid(z); return s * (1 - s)   # max = 0.25 at z=0

def tanh_fn(z):    return np.tanh(z)
def tanh_d(z):     return 1 - np.tanh(z)**2              # max = 1.0 at z=0

def relu(z):       return np.maximum(0, z)
def relu_d(z):     return (z > 0).astype(float)          # 0 or 1 — no vanishing

def leaky_relu(z, alpha=0.01): return np.where(z > 0, z, alpha * z)
def leaky_d(z, alpha=0.01):    return np.where(z > 0, 1.0, alpha)

def gelu(z):       return z * sigmoid(1.702 * z)         # approximation
def gelu_d(z):
    s = sigmoid(1.702 * z)
    return s + z * 1.702 * s * (1 - s)

activations = {
    'Sigmoid':     (sigmoid, sigmoid_d,   'tomato'),
    'Tanh':        (tanh_fn, tanh_d,      'steelblue'),
    'ReLU':        (relu,    relu_d,      'seagreen'),
    'Leaky ReLU':  (leaky_relu, leaky_d,  'orange'),
    'GELU':        (gelu,    gelu_d,      'purple'),
}

fig, axes = plt.subplots(2, 5, figsize=(20, 7))
for col, (name, (f, df, color)) in enumerate(activations.items()):
    axes[0, col].plot(z, f(z),  color=color, linewidth=2)
    axes[0, col].set_title(f'{name}\n(activation)')
    axes[0, col].axhline(0, color='k', alpha=0.3)
    axes[0, col].axvline(0, color='k', alpha=0.3)

    axes[1, col].plot(z, df(z), color=color, linewidth=2, linestyle='--')
    axes[1, col].set_title(f"Derivative\nmax={df(z).max():.2f}")
    axes[1, col].axhline(0, color='k', alpha=0.3)
    axes[1, col].set_ylim(-0.1, 1.2)

plt.suptitle('Activation Functions and Their Derivatives\n(Derivative max tells you: can gradients flow?)', fontsize=13)
plt.tight_layout()
plt.show()

# ── The vanishing gradient problem illustrated ─────────────────────────────────
print('Vanishing gradient problem:')
print('  Sigmoid derivative max = 0.25')
print('  After 10 layers: 0.25^10 =', round(0.25**10, 10), '-- gradient is essentially ZERO')
print('  Network stops learning in early layers')
print()
print('  ReLU derivative = 1.0 (for positive inputs)')
print('  After 10 layers: 1.0^10 =', 1.0**10, '-- gradient unchanged')
print('  This is why ReLU replaced sigmoid in deep networks (2012, AlexNet)')

# Dead ReLU problem
print('\nDead ReLU problem:')
print('  If z < 0, ReLU derivative = 0 -> that neuron never learns')
print('  Leaky ReLU fix: use 0.01*z instead of 0 for negative inputs')
print('  GELU fix: smooth approximation, used in transformers (BERT, GPT)')
```

> **Real-world scenario — AlexNet (2012) breakthrough:**
> Before 2012, deep networks used sigmoid activations throughout.
> Training 10+ layers failed because sigmoid's maximum derivative is 0.25.
> After 10 layers: `0.25^10 = 0.000000095` — gradients vanished, early layers learned nothing.
> AlexNet used ReLU instead. Training worked. It won ImageNet by a 10-point margin.

## CONCEPT 6 — Vanishing and Exploding Gradients

| Question | Answer |
|----------|--------|
| **What** | Vanishing: gradients shrink to ~0 in early layers. Exploding: gradients grow exponentially large. |
| **Why** | In deep networks, the chain rule multiplies many derivatives together — if each is < 1, the product shrinks; if > 1, it grows |
| **When** | Deep networks (10+ layers), RNNs processing long sequences |
| **Where** | Stage 2 — understanding why batch norm, residual connections, and gradient clipping exist |
| **Who** | Every DL engineer building or debugging deep networks |
| **How** | Vanishing: use ReLU, batch norm, residual connections. Exploding: gradient clipping. |
| **How often** | A concern in every deep network — these are the reasons for many standard architectural choices |
| **How much** | Understand the mechanism deeply. Know the fixes. |

```python
import numpy as np
import matplotlib.pyplot as plt

# ── Simulating gradient flow through many layers ────────────────────────────────
def simulate_gradient_flow(n_layers, activation='sigmoid'):
    # Simulate gradient magnitude as it flows backward through layers
    # Each layer multiplies the gradient by its local derivative

    np.random.seed(42)
    grad = 1.0   # start with gradient = 1 at output
    gradient_at_layer = [grad]

    for layer in range(n_layers):
        # Sample a random pre-activation value
        z = np.random.randn()

        if activation == 'sigmoid':
            s = 1 / (1 + np.exp(-z))
            local_deriv = s * (1 - s)         # max 0.25
        elif activation == 'tanh':
            local_deriv = 1 - np.tanh(z)**2   # max 1.0
        elif activation == 'relu':
            local_deriv = 1.0 if z > 0 else 0.0
        elif activation == 'leaky_relu':
            local_deriv = 1.0 if z > 0 else 0.01

        # Weight matrix contribution (random initialization)
        weight_scale = np.abs(np.random.randn() * 0.5)

        grad = grad * local_deriv * weight_scale
        gradient_at_layer.append(grad)

    return gradient_at_layer

fig, axes = plt.subplots(1, 2, figsize=(14, 5))

# Vanishing gradients (sigmoid in deep network)
n_layers = 30
for act, color in [('sigmoid','tomato'), ('tanh','orange'),
                   ('relu','steelblue'), ('leaky_relu','seagreen')]:
    grads = simulate_gradient_flow(n_layers, act)
    axes[0].plot(grads, label=act, color=color, linewidth=2)

axes[0].set_title('Gradient Magnitude Across Layers\n(sigmoid vanishes, ReLU flows)')
axes[0].set_xlabel('Layer (from output to input)')
axes[0].set_ylabel('|gradient|')
axes[0].legend()
axes[0].set_yscale('log')

# Exploding gradients (large weights)
grad_exp  = 1.0
grad_clip = 1.0
clip_val  = 1.0
grads_exp  = [grad_exp]
grads_clip = [grad_clip]

for _ in range(30):
    # Large weight scale -> gradient grows
    weight_scale = np.random.uniform(1.1, 1.3)
    grad_exp  *= weight_scale
    grad_clip *= weight_scale
    # Gradient clipping
    if abs(grad_clip) > clip_val:
        grad_clip = np.sign(grad_clip) * clip_val
    grads_exp.append(grad_exp)
    grads_clip.append(grad_clip)

axes[1].plot(grads_exp,  label='No clipping (explodes)',  color='tomato',    linewidth=2)
axes[1].plot(grads_clip, label='Gradient clipping (stable)', color='steelblue', linewidth=2)
axes[1].set_title('Exploding Gradients + Clipping Fix')
axes[1].set_xlabel('Layer'); axes[1].set_ylabel('|gradient|')
axes[1].legend()

plt.tight_layout()
plt.show()

# Solutions summary
print('VANISHING GRADIENT SOLUTIONS:')
print('  1. ReLU activation    -> derivative stays 1 for positive inputs')
print('  2. Residual connections -> gradients flow directly (skip layers)')
print('  3. Batch normalization -> keeps activations in good range per layer')
print('  4. Proper weight init  -> Xavier/He initialization keeps gradients ~1')
print()
print('EXPLODING GRADIENT SOLUTIONS:')
print('  1. Gradient clipping   -> torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)')
print('  2. Lower learning rate -> smaller steps, less chance of explosion')
print('  3. Batch normalization -> stabilizes activations')
```

> **Real-world scenario — Training LSTMs on long sequences:**
> An LSTM processing 500-token sequences multiplies gradients across 500 time steps.
> Without gradient clipping, the gradient from token 1 (after 500 multiplications) becomes either zero or infinity.
> Every major NLP training script includes `clip_grad_norm_` as standard practice.
> The famous GPT training code by Andrej Karpathy includes gradient clipping on line 1 of the training loop.

> **Used in:** Stage 2 — Residual connections (ResNet), Layer Norm; Stage 3 — LLM training stability

## CONCEPT 7 — PyTorch Autograd: Calculus on Autopilot

| Question | Answer |
|----------|--------|
| **What** | PyTorch's automatic differentiation engine — records operations in a graph and runs chain rule backward automatically |
| **Why** | You never manually compute derivatives in practice — autograd does it for every parameter |
| **When** | Every neural network training loop: `loss.backward()` |
| **Where** | The machinery behind every PyTorch/TensorFlow model |
| **Who** | Every ML engineer using deep learning frameworks |
| **How** | Tensors with `requires_grad=True` track their computation history. `backward()` walks the graph in reverse. |
| **How often** | Every training iteration |
| **How much** | Know how to use it. Understand what it is doing. Know common pitfalls (detach, zero_grad). |

```python
import torch
import torch.nn as nn
import matplotlib.pyplot as plt
import numpy as np

# ── Manual autograd — scalar example ──────────────────────────────────────────
w = torch.tensor(2.0, requires_grad=True)   # track this variable
x = torch.tensor(3.0)                       # input (fixed)

# Forward computation
z = w * x       # z = 6.0
L = z ** 2      # L = 36.0  (our "loss")

print(f'Forward: w={w.item()}, x={x.item()}, z={z.item()}, L={L.item()}')

# Backward: compute dL/dw
L.backward()
print(f'dL/dw = {w.grad.item()}')   # = 2 * z * x = 2 * 6 * 3 = 36? No: dL/dz * dz/dw = 2z * x = 2*6*3=36
# Actually: dL/dw = dL/dz * dz/dw = (2*z) * x = 12 * 3 = 36
# Wait: z=w*x=6, dL/dz = 2*z = 12, dz/dw = x = 3 -> dL/dw = 12*3 = 36
print(f'Manual: dL/dw = 2*z*x = {2*6*3}')

# ── Full neural network with autograd ──────────────────────────────────────────
torch.manual_seed(42)
np.random.seed(42)

# Dataset: linear relationship y = 3x + 1.5
n_pts = 100
X_t = torch.tensor(np.random.randn(n_pts, 1).astype(np.float32))
y_t = torch.tensor((3 * X_t.numpy() + 1.5 + 0.3 * np.random.randn(n_pts, 1)).astype(np.float32))

# Simple linear model
model     = nn.Linear(1, 1)   # y = w*x + b
optimizer = torch.optim.SGD(model.parameters(), lr=0.1)
criterion = nn.MSELoss()

losses = []
for epoch in range(100):
    # Forward pass
    y_pred = model(X_t)
    loss   = criterion(y_pred, y_t)
    losses.append(loss.item())

    # Backward pass — autograd computes ALL gradients via chain rule
    optimizer.zero_grad()    # CRITICAL: clear previous gradients
    loss.backward()          # chain rule runs here -> fills .grad for all parameters
    optimizer.step()         # w = w - lr * w.grad

print(f'Trained: w={model.weight.item():.4f} (true=3.0)  b={model.bias.item():.4f} (true=1.5)')

# Show what backward() computed
X_sample = torch.tensor([[1.5]])
y_sample  = torch.tensor([[6.0]])
optimizer.zero_grad()
loss_s = criterion(model(X_sample), y_sample)
loss_s.backward()
print(f'\nGradients after backward():')
print(f'  dL/dw = {model.weight.grad.item():.6f}')
print(f'  dL/db = {model.bias.grad.item():.6f}')

# ── Common pitfalls ─────────────────────────────────────────────────────────────
print('\nCommon autograd pitfalls:')
print('1. Forgetting zero_grad() -> gradients ACCUMULATE across iterations -> wrong update')
w2 = torch.tensor(1.0, requires_grad=True)
for i in range(3):
    L2 = (w2 * 2) ** 2
    L2.backward()
    print(f'   iter {i}: grad = {w2.grad.item():.1f}  <- accumulates without zero_grad!')

print('2. Using .detach() to stop gradient tracking:')
z_tracked = w2 * 3
z_stopped = z_tracked.detach()  # no longer in computation graph
print(f'   z_tracked.requires_grad = {z_tracked.requires_grad}')
print(f'   z_stopped.requires_grad = {z_stopped.requires_grad}')

# Loss curve
fig, ax = plt.subplots(figsize=(9, 4))
ax.plot(losses, 'steelblue', linewidth=2)
ax.set_title('PyTorch Training Loss — Autograd + SGD')
ax.set_xlabel('Epoch'); ax.set_ylabel('MSE Loss')
plt.tight_layout()
plt.show()
```

> **Real-world scenario — PyTorch training loop anatomy:**
> Every professional PyTorch training loop has exactly these 4 lines inside the loop:
> 1. `output = model(batch)` — forward pass, autograd records operations
> 2. `loss = loss_fn(output, labels)` — compute scalar loss
> 3. `optimizer.zero_grad()` + `loss.backward()` — clear old gradients, compute new ones via chain rule
> 4. `optimizer.step()` — update all weights: `w -= lr * w.grad`
> Understanding calculus tells you what each of these lines actually does.

> **Used in:** Stage 2 — Every PyTorch model training; every neural network tutorial

## CONCEPT 8 — Adam Optimizer: Adaptive Learning Rates

| Question | Answer |
|----------|--------|
| **What** | An optimizer that adapts the learning rate per parameter using estimates of first and second moments of gradients |
| **Why** | SGD uses the same learning rate for all parameters — some need large steps, some need small. Adam adapts automatically. |
| **When** | Default optimizer for most neural networks, LLMs, transformers |
| **Where** | Stage 2 — PyTorch `torch.optim.Adam`; used in every major model |
| **Who** | Every DL engineer — Adam is the standard starting optimizer |
| **How** | Keeps running averages of gradient (m) and gradient squared (v); scales each weight's update by `m / sqrt(v)` |
| **How often** | Most training runs use Adam or its variant (AdamW) |
| **How much** | Know when to use Adam vs SGD, understand what `beta1`, `beta2`, `eps` do |

```python
import numpy as np
import matplotlib.pyplot as plt

# ── Adam from scratch ──────────────────────────────────────────────────────────
# Adam update rule:
#   m = beta1*m + (1-beta1)*g          <- first moment (mean of gradients)
#   v = beta2*v + (1-beta2)*g^2        <- second moment (variance of gradients)
#   m_hat = m / (1 - beta1^t)          <- bias correction
#   v_hat = v / (1 - beta2^t)          <- bias correction
#   w = w - lr * m_hat / (sqrt(v_hat) + eps)

def adam_step(w, g, m, v, t, lr=0.001, beta1=0.9, beta2=0.999, eps=1e-8):
    m = beta1 * m + (1 - beta1) * g
    v = beta2 * v + (1 - beta2) * g**2
    m_hat = m / (1 - beta1**t)
    v_hat = v / (1 - beta2**t)
    w = w - lr * m_hat / (np.sqrt(v_hat) + eps)
    return w, m, v

# Compare SGD vs Adam on a pathological loss surface
# Rosenbrock function: f(x,y) = (1-x)^2 + 100(y-x^2)^2
def rosenbrock(x, y): return (1-x)**2 + 100*(y - x**2)**2
def rosenbrock_grad(x, y):
    gx = -2*(1-x) - 400*x*(y - x**2)
    gy = 200*(y - x**2)
    return np.array([gx, gy])

# SGD run
w_sgd = np.array([-1.5, 1.5])
lr_s  = 0.001
traj_sgd = [w_sgd.copy()]
for _ in range(5000):
    g = rosenbrock_grad(*w_sgd)
    w_sgd = w_sgd - lr_s * g
    traj_sgd.append(w_sgd.copy())
traj_sgd = np.array(traj_sgd)

# Adam run
w_adam = np.array([-1.5, 1.5])
m_adam = np.zeros(2)
v_adam = np.zeros(2)
traj_adam = [w_adam.copy()]
for t in range(1, 5001):
    g = rosenbrock_grad(*w_adam)
    w_adam, m_adam, v_adam = adam_step(w_adam, g, m_adam, v_adam, t, lr=0.01)
    traj_adam.append(w_adam.copy())
traj_adam = np.array(traj_adam)

print(f'SGD  final: ({w_sgd[0]:.4f}, {w_sgd[1]:.4f})  target=(1,1)')
print(f'Adam final: ({w_adam[0]:.4f}, {w_adam[1]:.4f})  target=(1,1)')

# Visualize trajectories
x_r = np.linspace(-2, 2, 300); y_r = np.linspace(-0.5, 2.5, 300)
X_r, Y_r = np.meshgrid(x_r, y_r)
Z_r = np.log(1 + rosenbrock(X_r, Y_r))   # log scale for visibility

fig, axes = plt.subplots(1, 2, figsize=(14, 5))
for ax, traj, name, color in [
    (axes[0], traj_sgd,  'SGD',  'tomato'),
    (axes[1], traj_adam, 'Adam', 'steelblue'),
]:
    ax.contourf(X_r, Y_r, Z_r, levels=30, cmap='viridis', alpha=0.7)
    ax.plot(traj[:,0], traj[:,1], color=color, linewidth=1, alpha=0.7, label=name)
    ax.scatter(*traj[0],  s=80, color='white',  zorder=5, label='start')
    ax.scatter(*traj[-1], s=80, color='yellow', zorder=5, label='end')
    ax.scatter(1, 1,      s=80, color='lime',   zorder=5, marker='*', label='minimum (1,1)')
    ax.set_title(f'{name} on Rosenbrock — 5000 steps')
    ax.legend(fontsize=8)
    ax.set_xlim(-2, 2); ax.set_ylim(-0.5, 2.5)

plt.suptitle('SGD vs Adam — Adam adapts learning rate per dimension', fontsize=12)
plt.tight_layout()
plt.show()

# AdamW — Adam + weight decay (the standard for LLMs)
print('\nAdam hyperparameters:')
print('  beta1=0.9   -> how quickly to forget old gradient direction')
print('  beta2=0.999 -> how quickly to forget old gradient variance')
print('  eps=1e-8    -> prevents division by zero when gradient variance is tiny')
print('  AdamW       -> adds weight decay (L2 regularization) correctly')
print('  -> default for BERT, GPT, and most transformers')
```

> **Real-world scenario — Why Adam dominates deep learning:**
> Some weights in a neural network get large gradients (weights near the output).
> Others get tiny gradients (weights near the input — vanishing gradient effect).
> SGD uses the same learning rate for both → too large for some, too small for others.
> Adam divides each weight's gradient by the history of that gradient's variance.
> Weights with consistently large gradients get a smaller effective learning rate.
> Weights with small, noisy gradients get a larger effective learning rate.
> This is why Adam trains models 5-10× faster than SGD on complex tasks.

> **Used in:** Stage 2 — `torch.optim.Adam` and `torch.optim.AdamW`; Stage 3 — LLM training

```python
import torch
import torch.nn as nn
import numpy as np
import matplotlib.pyplot as plt
from sklearn.datasets import make_moons
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import train_test_split

torch.manual_seed(42)
np.random.seed(42)

# Dataset
X_np, y_np  = make_moons(n_samples=600, noise=0.2, random_state=42)
X_np        = StandardScaler().fit_transform(X_np)
X_tr, X_te, y_tr, y_te = train_test_split(X_np, y_np, test_size=0.2)

X_train = torch.FloatTensor(X_tr)
y_train = torch.FloatTensor(y_tr).unsqueeze(1)
X_test  = torch.FloatTensor(X_te)
y_test  = torch.FloatTensor(y_te).unsqueeze(1)

# Model: 2 -> 16 -> 16 -> 1
model = nn.Sequential(
    nn.Linear(2, 16),
    nn.ReLU(),            # derivative = 1 if z>0, else 0
    nn.Linear(16, 16),
    nn.ReLU(),
    nn.Linear(16, 1),
    nn.Sigmoid()          # derivative = s*(1-s)
)

optimizer = torch.optim.Adam(model.parameters(), lr=0.01)
criterion = nn.BCELoss()   # Binary cross-entropy = -log likelihood

print('Model parameters:', sum(p.numel() for p in model.parameters()))

# Training loop — all calculus concepts combined
train_losses, test_losses = [], []
grad_norms = []

for epoch in range(200):
    model.train()

    # FORWARD PASS (matrix multiplies, activations)
    y_pred = model(X_train)
    loss   = criterion(y_pred, y_train)

    # BACKWARD PASS (chain rule via autograd)
    optimizer.zero_grad()   # clear accumulated gradients
    loss.backward()         # compute dL/dw for ALL parameters

    # GRADIENT CLIPPING (prevent explosion)
    grad_norm = torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
    grad_norms.append(grad_norm)

    # WEIGHT UPDATE (Adam: adaptive gradient descent)
    optimizer.step()

    train_losses.append(loss.item())

    # Validation
    model.eval()
    with torch.no_grad():   # no need to track gradients during eval
        test_loss = criterion(model(X_test), y_test).item()
        test_losses.append(test_loss)

# Final accuracy
model.eval()
with torch.no_grad():
    train_acc = ((model(X_train) > 0.5).float() == y_train).float().mean().item()
    test_acc  = ((model(X_test)  > 0.5).float() == y_test ).float().mean().item()

print(f'Train accuracy: {train_acc:.4f}  |  Test accuracy: {test_acc:.4f}')

# Visualize
fig, axes = plt.subplots(1, 3, figsize=(18, 5))

axes[0].plot(train_losses, label='Train Loss', color='steelblue')
axes[0].plot(test_losses,  label='Test Loss',  color='tomato', linestyle='--')
axes[0].set_title('Loss Curves — Cross-Entropy (Negative Log-Likelihood)')
axes[0].set_xlabel('Epoch'); axes[0].legend()

axes[1].plot(grad_norms, color='seagreen', alpha=0.7)
axes[1].axhline(1.0, color='red', linestyle='--', label='clip threshold')
axes[1].set_title('Gradient Norms — Before Clipping\n(shows training stability)')
axes[1].set_xlabel('Epoch'); axes[1].legend()

# Decision boundary
xx, yy = np.meshgrid(np.linspace(-3, 3, 100), np.linspace(-3, 3, 100))
grid   = torch.FloatTensor(np.c_[xx.ravel(), yy.ravel()])
with torch.no_grad():
    probs = model(grid).numpy().reshape(xx.shape)
axes[2].contourf(xx, yy, probs, levels=50, cmap='RdBu', alpha=0.7)
axes[2].scatter(X_te[:,0], X_te[:,1], c=y_te, cmap='RdBu', edgecolors='k', s=20)
axes[2].set_title(f'Learned Decision Boundary\n(test acc={test_acc:.3f})')

plt.tight_layout()
plt.show()
```

```python
import torch
import torch.nn as nn
import numpy as np
import matplotlib.pyplot as plt

torch.manual_seed(42)
np.random.seed(42)

# Generate regression data
n = 300
X_np = np.random.randn(n, 4).astype(np.float32)
y_np = (X_np[:, 0] * 2 - X_np[:, 1] + 0.5 * X_np[:, 2] + np.random.randn(n) * 0.3).astype(np.float32)

X_t = torch.FloatTensor(X_np)
y_t = torch.FloatTensor(y_np).unsqueeze(1)

def build_model():
    return nn.Sequential(
        nn.Linear(4, 32), nn.Tanh(),
        nn.Linear(32, 32), nn.Tanh(),
        nn.Linear(32, 1)
    )

def train(model, optimizer, clip=None, n_epochs=100):
    criterion = nn.MSELoss()
    losses = []
    grad_norms = []
    for epoch in range(n_epochs):
        y_pred = model(X_t)
        loss   = criterion(y_pred, y_t)
        optimizer.zero_grad()
        loss.backward()
        gn = torch.nn.utils.clip_grad_norm_(model.parameters(),
                                             max_norm=clip if clip else float('inf'))
        grad_norms.append(gn)
        optimizer.step()
        losses.append(loss.item())
    return losses, grad_norms

# ── BROKEN: too high learning rate -> exploding gradients ─────────────────────

losses_broken, norms_broken = train(model_broken, opt_broken)

# ── FIX 1: lower learning rate ─────────────────────────────────────────────────
model_fix1  = build_model()
opt_fix1    = torch.optim.SGD(model_fix1.parameters(), lr=0.01)
losses_fix1, norms_fix1 = train(model_fix1, opt_fix1)

# ── FIX 2: gradient clipping ──────────────────────────────────────────────────
model_fix2  = build_model()
opt_fix2    = torch.optim.SGD(model_fix2.parameters(), lr=0.1)
losses_fix2, norms_fix2 = train(model_fix2, opt_fix2, clip=1.0)

# ── FIX 3: Adam (adapts learning rate automatically) ──────────────────────────
model_fix3  = build_model()
opt_fix3    = torch.optim.Adam(model_fix3.parameters(), lr=0.01)
losses_fix3, norms_fix3 = train(model_fix3, opt_fix3)

fig, axes = plt.subplots(1, 2, figsize=(14, 5))

axes[0].plot(losses_broken, label='SGD lr=1.0 (BROKEN)',  color='tomato',    linewidth=2)
axes[0].plot(losses_fix1,   label='SGD lr=0.01 (fixed)',  color='steelblue', linewidth=2)

axes[0].set_title('Loss Curves — Diagnosing Exploding Gradients')

axes[1].plot(norms_broken, label='SGD broken',  color='tomato',    linewidth=2, alpha=0.7)

axes[1].plot(norms_fix3,   label='Adam',        color='seagreen',  linewidth=2, alpha=0.7)
axes[1].set_title('Gradient Norm Over Time\n(high norm = explosion risk)')
axes[1].set_xlabel('Epoch'); axes[1].set_ylabel('Grad Norm')
axes[1].set_ylim(0, 50); axes[1].legend()
```
