## Key Takeaways — Read This Before Any Project

1. **matplotlib = the engine** — all other libraries build on top of it. Learn it first.
2. **seaborn = matplotlib with better defaults** — use seaborn for EDA, matplotlib when you need full control
3. **Always label your plots** — `title`, `xlabel`, `ylabel`. A plot without labels is meaningless to anyone else (including future you)
4. **`plt.tight_layout()`** — call this before `plt.show()` every time or subplots overlap
5. **`sns.heatmap(df.corr())`** — the single most useful plot in EDA. Shows which features are related to each other and to the target
6. **`sns.pairplot(df)`** — one line to see ALL feature relationships at once. Run this early on every dataset
7. **plotly = interactive** — use when you need to zoom, hover, or share plots in a web app or dashboard

## SECTION 1 — matplotlib Basics

### The Figure and Axes Model ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | matplotlib has two objects: `Figure` (the whole canvas) and `Axes` (one individual plot). |
| **Why** | Understanding this distinction lets you control exactly where things go — especially when making multi-plot layouts. |
| **When** | Every time you make a plot. |
| **Where** | All stages — plotting training curves, EDA, model evaluation. |
| **Who** | Every ML engineer for visualizing experiments and results. |
| **How** | `plt.subplots()` returns a `Figure` and one or more `Axes`. You draw on the `Axes`. |

```python
import matplotlib.pyplot as plt
import numpy as np

# Simple way — pyplot handles figure/axes automatically
plt.plot([1, 2, 3], [4, 5, 6])
plt.title('My Plot')
plt.xlabel('X axis')
plt.ylabel('Y axis')
plt.show()

# Explicit way — use this for anything beyond one plot
fig, ax = plt.subplots(figsize=(8, 5))   # figsize = (width, height) in inches
ax.plot([1, 2, 3], [4, 5, 6])
ax.set_title('My Plot')
ax.set_xlabel('X axis')
ax.set_ylabel('Y axis')
plt.tight_layout()
plt.show()
```

> **Rule:** For single plots, `plt.plot()` is fine. For anything with multiple subplots or tight control, always use `fig, ax = plt.subplots()`. The explicit form is what you'll use in professional code.

### `plt.plot()` — Line Chart ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | Draws a line connecting data points in sequence. |
| **Why** | Shows trends — how a value changes across time or ordered steps. |
| **When** | Training loss over epochs, accuracy over time, any ordered sequence. |
| **Where** | Stage 2 — plotting training curves every time you train a model. |

```python
epochs = np.arange(1, 21)
train_loss = np.exp(-epochs * 0.2) + np.random.randn(20) * 0.02
val_loss   = np.exp(-epochs * 0.15) + np.random.randn(20) * 0.03 + 0.1

fig, ax = plt.subplots(figsize=(9, 5))
ax.plot(epochs, train_loss, label='Training Loss',   color='blue',   linewidth=2)
ax.plot(epochs, val_loss,   label='Validation Loss', color='orange', linewidth=2, linestyle='--')
ax.set_title('Training vs Validation Loss')
ax.set_xlabel('Epoch')
ax.set_ylabel('Loss')
ax.legend()
ax.grid(True, alpha=0.3)
plt.tight_layout()
plt.show()
```

> **Used in Stage 2:** Every time you train a neural network, you plot training vs validation loss. If validation loss starts rising while training loss keeps dropping — that's overfitting. This plot tells you when to stop training.

### `plt.scatter()` — Scatter Plot ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | Plots individual data points as dots — one dot per sample. |
| **Why** | Shows the relationship between two numeric features. Do they correlate? Are there clusters? Any outliers? |
| **When** | EDA — understanding feature relationships. After training — comparing predicted vs actual values. |
| **Where** | Stage 1 (EDA), Stage 2 (visualizing embeddings with t-SNE/UMAP). |

```python
np.random.seed(42)
x = np.random.randn(200)
y = 2 * x + np.random.randn(200) * 0.8    # correlated with x + noise
colors = np.where(y > 0, 'steelblue', 'tomato')  # color by class

fig, ax = plt.subplots(figsize=(7, 5))
ax.scatter(x, y, c=colors, alpha=0.6, s=40, edgecolors='white', linewidth=0.5)
ax.set_title('Feature Relationship')
ax.set_xlabel('Feature A')
ax.set_ylabel('Feature B')
ax.axhline(0, color='black', linewidth=0.8, linestyle='--')
ax.axvline(0, color='black', linewidth=0.8, linestyle='--')
plt.tight_layout()
plt.show()
```

> **Used in Stage 1:** After training a regression model, plot `actual vs predicted` as a scatter. Perfect model = all dots on a diagonal line. Spread around the line = model error you can see.

### `plt.hist()` — Histogram ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | Shows the distribution of a single numeric column — how values are spread. |
| **Why** | Before normalizing a feature, you need to see its distribution. Is it normal? Skewed? Bimodal? This determines which normalization to use. |
| **When** | EDA — inspect every numeric feature before modeling. |
| **Where** | Stage 1 — standard first step before preprocessing. |

```python
np.random.seed(42)
normal_data = np.random.randn(1000)
skewed_data = np.random.exponential(2, 1000)

fig, axes = plt.subplots(1, 2, figsize=(12, 4))

axes[0].hist(normal_data, bins=40, color='steelblue', edgecolor='white', alpha=0.8)
axes[0].set_title('Normal Distribution')
axes[0].set_xlabel('Value')
axes[0].set_ylabel('Frequency')

axes[1].hist(skewed_data, bins=40, color='tomato', edgecolor='white', alpha=0.8)
axes[1].set_title('Skewed Distribution (Exponential)')
axes[1].set_xlabel('Value')
axes[1].set_ylabel('Frequency')

plt.tight_layout()
plt.show()
```

> **Why distributions matter for ML:** A skewed feature (like house price or income) should be log-transformed before training. A normal feature works fine with z-score normalization. You cannot make this decision without seeing the histogram first.

---

### `plt.bar()` — Bar Chart ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | Compares values across categories using rectangular bars. |
| **Why** | Shows class distribution (are your classes imbalanced?), feature importance, model comparison. |
| **When** | Class balance check before training, comparing model performance across categories. |
| **Where** | Stage 1 — class imbalance is a critical issue in ML. |

```python
categories = ['Eng', 'Sales', 'HR', 'Product']
counts      = [45, 30, 15, 10]
colors      = ['steelblue', 'seagreen', 'tomato', 'gold']

fig, axes = plt.subplots(1, 2, figsize=(12, 4))

# Vertical bar
axes[0].bar(categories, counts, color=colors, edgecolor='white')
axes[0].set_title('Employee Count by Department')
axes[0].set_ylabel('Count')

# Horizontal bar — better for long category names
axes[1].barh(categories, counts, color=colors, edgecolor='white')
axes[1].set_title('Employee Count by Department (Horizontal)')
axes[1].set_xlabel('Count')

plt.tight_layout()
plt.show()
```

> **Used in Stage 1:** Before training a classifier, always plot class distribution with a bar chart. If one class has 95% of samples and another has 5%, your model will just predict the majority class. You need to know this BEFORE training.

### `plt.subplots()` — Multiple Plots Grid ⭐⭐⭐ 🧠 Pattern, 📖 Args

| | |
|---|---|
| **What** | Creates a grid of multiple plots in one figure. |
| **Why** | In EDA you need to inspect many features at once. Separate figures are hard to compare. A grid lets you see everything side by side. |
| **When** | Inspecting multiple features in one view, comparing model results across settings. |

```python
np.random.seed(42)
fig, axes = plt.subplots(2, 3, figsize=(15, 8))   # 2 rows, 3 columns

feature_names = ['Age', 'Salary', 'Experience', 'Score', 'Bonus', 'Tax']
for i, ax in enumerate(axes.flatten()):            # flatten() converts 2D grid to 1D list
    data = np.random.randn(200) * (i + 1) * 10
    ax.hist(data, bins=30, color=f'C{i}', edgecolor='white', alpha=0.8)
    ax.set_title(f'{feature_names[i]} Distribution')
    ax.set_xlabel('Value')
    ax.set_ylabel('Count')

plt.suptitle('Feature Distributions', fontsize=14, y=1.02)
plt.tight_layout()
plt.show()
```

> **Pattern to remember:** `fig, axes = plt.subplots(rows, cols)` then `axes.flatten()` to iterate over all subplots in order. This pattern appears in every multi-feature EDA.

### Why seaborn Exists ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | A statistical visualization library built on top of matplotlib — better default styles and smarter chart types. |
| **Why** | matplotlib is powerful but verbose. seaborn writes less code, looks better by default, and understands pandas DataFrames natively. |
| **When** | EDA — exploring datasets, understanding distributions and relationships. |
| **Who** | Data scientists and ML engineers for all exploratory analysis. |

```python
import pandas as pd

# Set style once at top of notebook
sns.set_theme(style='whitegrid', palette='muted')

# seaborn understands DataFrames directly — no need to extract columns manually
df = pd.DataFrame({
    'salary': np.random.randint(50000, 120000, 100),
    'age':    np.random.randint(22, 55, 100),
    'dept':   np.random.choice(['Eng', 'Sales', 'HR'], 100),
    'score':  np.random.uniform(60, 100, 100)
})
```

### `sns.histplot()` — Distribution with KDE ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | Histogram with an optional KDE (Kernel Density Estimate) smooth curve overlaid. |
| **Why** | Better than `plt.hist()` — shows the shape of the distribution more clearly. Also works directly with DataFrame columns. |
| **When** | Every time you inspect a numeric feature's distribution during EDA. |

```python
fig, axes = plt.subplots(1, 2, figsize=(13, 4))

# Basic
sns.histplot(df['salary'], bins=30, kde=True, ax=axes[0], color='steelblue')
axes[0].set_title('Salary Distribution')

# Grouped by category — overlapping distributions
sns.histplot(df, x='salary', hue='dept', kde=True, ax=axes[1], alpha=0.5)
axes[1].set_title('Salary Distribution by Department')

plt.tight_layout()
plt.show()
```

> **`kde=True` means:** Draw a smooth curve over the histogram showing the estimated probability distribution. This helps you see if data is normal, skewed, or bimodal — critical for choosing the right preprocessing.

---

### `sns.boxplot()` — Outliers and Spread ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | Shows median, quartiles, and outliers of a distribution — grouped by a category. |
| **Why** | Histograms show one group. Box plots compare many groups side by side and highlight outliers clearly. |
| **When** | Comparing a numeric feature across categories, spotting outliers before training. |
| **Where** | Stage 1 — outlier detection and feature comparison across classes. |

```python
fig, axes = plt.subplots(1, 2, figsize=(13, 5))

sns.boxplot(data=df, x='dept', y='salary', ax=axes[0], palette='muted')
axes[0].set_title('Salary by Department')

sns.boxplot(data=df, x='dept', y='score', ax=axes[1], palette='pastel')
axes[1].set_title('Score by Department')

plt.tight_layout()
plt.show()
```

> **Reading a box plot:**
> - The **box** = middle 50% of data (IQR)
> - The **line inside the box** = median
> - The **whiskers** = range up to 1.5×IQR
> - **Dots beyond whiskers** = outliers — these are the values you should investigate before training

### `sns.heatmap()` — Correlation Matrix ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | A color-coded grid showing the correlation between every pair of numeric features. |
| **Why** | Before training, you need to know which features are related. If two features are 95% correlated, you only need one — the other adds noise. Also shows which features correlate with your target. |
| **When** | First step in feature selection during EDA. Non-negotiable. |
| **Where** | Stage 1 — before every classification or regression model. |
| **How often** | Every project with multiple numeric features. |

```python
# Create a more realistic dataset
np.random.seed(42)
n = 200
df_corr = pd.DataFrame({
    'age':        np.random.randint(22, 60, n),
    'salary':     np.random.randint(40000, 150000, n),
    'experience': np.random.randint(1, 35, n),
    'score':      np.random.uniform(50, 100, n),
    'bonus':      np.random.randint(1000, 20000, n),
})
df_corr['salary']     = df_corr['salary'] + df_corr['experience'] * 2000  # introduce correlation
df_corr['bonus']      = df_corr['salary'] * 0.1 + np.random.randn(n) * 500

corr_matrix = df_corr.corr()

fig, ax = plt.subplots(figsize=(8, 6))
sns.heatmap(
    corr_matrix,
    annot=True,          # show the number in each cell
    fmt='.2f',           # 2 decimal places
    cmap='coolwarm',     # red=positive, blue=negative correlation
    center=0,            # 0 correlation = white
    square=True,
    linewidths=0.5,
    ax=ax
)
ax.set_title('Feature Correlation Matrix')
plt.tight_layout()
plt.show()
```

> **How to read this:** Values close to `1.0` = strong positive correlation. Close to `-1.0` = strong negative. Close to `0` = no relationship.
>
> **What to look for:**
> - High correlation between features → consider dropping one (reduces noise)
> - High correlation between a feature and your target → this feature is predictive — keep it
> - All features near 0 with the target → you might need feature engineering

> **Used in Stage 1:** Before training, run `sns.heatmap(df.corr())`. It tells you which features matter and which are redundant. This single plot drives feature selection decisions.

### `sns.pairplot()` — All Features vs All Features ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | Creates a grid of scatter plots between every pair of features, with histograms on the diagonal. |
| **Why** | One line gives you a complete picture of all feature relationships at once. |
| **When** | Early EDA on a new dataset — run this once to get the big picture. |
| **Where** | Stage 1 — first exploratory step on any structured dataset. |

```python
# Color by a category to see class separation
sns.pairplot(
    df_corr.assign(dept=np.random.choice(['Eng', 'Sales', 'HR'], len(df_corr))),
    hue='dept',           # color points by category
    diag_kind='kde',      # KDE on diagonal instead of histogram
    plot_kws={'alpha': 0.5, 's': 20}
)
plt.suptitle('Pairplot — All Feature Relationships', y=1.01)
plt.show()
```

> **When to use:** Run `sns.pairplot(df)` at the very start of any new dataset. It takes 5 seconds and shows you everything in one view. If you see clear separation between colored groups in any scatter plot, that feature is highly predictive.

### `sns.countplot()` — Category Frequencies ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | Bar chart that automatically counts occurrences of each category. |
| **Why** | Essential for checking class imbalance before training a classifier. An imbalanced dataset silently produces a bad model. |
| **When** | Always — before training any classification model. |

```python
fig, axes = plt.subplots(1, 2, figsize=(13, 4))

# Balanced classes
df_balanced = pd.DataFrame({'label': np.random.choice(['Cat', 'Dog', 'Bird'], 300)})
sns.countplot(data=df_balanced, x='label', ax=axes[0], palette='pastel')

df_imbal = pd.DataFrame({'label': np.random.choice(['Fraud', 'Legit'], 300, p=[0.05, 0.95])})
```

> **Used in Stage 1:** Before training a fraud detection model, run `sns.countplot(df['label'])`. Seeing this plot forces you to handle the imbalance before training.

### Confusion Matrix Heatmap ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | A grid showing true vs predicted labels for a classifier — how many it got right and where it made mistakes. |
| **Why** | Accuracy alone is misleading. A confusion matrix shows WHICH classes your model confuses. |
| **When** | After training any classification model — always. |
| **Where** | Stage 1 onwards — every classification task. |

```python
from sklearn.metrics import confusion_matrix

# Simulated true labels and predictions
y_true = np.array([0,0,0,1,1,1,2,2,2,0,1,2,0,1,2])
y_pred = np.array([0,0,1,1,1,2,2,2,2,0,0,2,0,1,1])

cm = confusion_matrix(y_true, y_pred)

fig, ax = plt.subplots(figsize=(6, 5))
sns.heatmap(
    cm,
    annot=True,
    fmt='d',
    cmap='Blues',
    xticklabels=['Class 0', 'Class 1', 'Class 2'],
    yticklabels=['Class 0', 'Class 1', 'Class 2'],
    ax=ax
)
ax.set_title('Confusion Matrix')
ax.set_xlabel('Predicted Label')
ax.set_ylabel('True Label')
plt.tight_layout()
plt.show()
```

> **How to read it:** Diagonal = correct predictions. Off-diagonal = mistakes. If cell (1,2) is high, your model often predicts Class 2 when the true label is Class 1 — you need to investigate why.

---

### Training Curves — Loss and Accuracy ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | Line plots of training and validation loss/accuracy over epochs during model training. |
| **Why** | The most important diagnostic tool in deep learning. It tells you if the model is learning, overfitting, or underfitting — before you waste compute. |
| **When** | After every training run — always. |
| **Where** | Stage 2 onwards — every neural network training session. |

```python
np.random.seed(42)
epochs = np.arange(1, 51)

# Simulate an overfitting scenario
train_loss = 1.0 / (1 + epochs * 0.1) + np.random.randn(50) * 0.01
val_loss   = 1.0 / (1 + epochs * 0.07) + 0.05 * np.log(1 + epochs * 0.05) + np.random.randn(50) * 0.02

train_acc  = 1 - train_loss / 2
val_acc    = 1 - val_loss / 2

fig, axes = plt.subplots(1, 2, figsize=(14, 5))

# Loss
axes[0].plot(epochs, train_loss, label='Train Loss', color='blue',   linewidth=2)
axes[0].plot(epochs, val_loss,   label='Val Loss',   color='orange', linewidth=2, linestyle='--')
axes[0].axvline(x=30, color='red', linestyle=':', label='Overfitting starts')
axes[0].set_title('Loss Curves')
axes[0].set_xlabel('Epoch')
axes[0].set_ylabel('Loss')
axes[0].legend()
axes[0].grid(True, alpha=0.3)

# Accuracy
axes[1].plot(epochs, train_acc, label='Train Accuracy', color='blue',   linewidth=2)
axes[1].plot(epochs, val_acc,   label='Val Accuracy',   color='orange', linewidth=2, linestyle='--')
axes[1].set_title('Accuracy Curves')
axes[1].set_xlabel('Epoch')
axes[1].set_ylabel('Accuracy')
axes[1].legend()
axes[1].grid(True, alpha=0.3)

plt.suptitle('Training Curves — Overfitting Detected at Epoch ~30', fontsize=13)
plt.tight_layout()
plt.show()
```

> **Reading training curves:**
> - Train loss ↓ + Val loss ↓ together → model is learning correctly
> - Train loss ↓ + Val loss starts ↑ → **overfitting** — model memorizing training data
> - Both losses plateau early → **underfitting** — model too simple or learning rate too low

## SECTION 4 — plotly (Interactive Visualization)

### Why plotly ⭐⭐ 📖

| | |
|---|---|
| **What** | Creates interactive charts — you can zoom, pan, hover over data points, and filter. |
| **Why** | matplotlib/seaborn produce static images. plotly produces interactive HTML — great for exploring large datasets, dashboards, and sharing results. |
| **When** | When presenting to stakeholders, building monitoring dashboards, or exploring dense scatter plots where you need to hover over individual points. |
| **Where** | Stage 7 (MLOps dashboards), production monitoring, Jupyter notebooks for exploration. |

```python
import plotly.express as px

# Interactive scatter — hover to see individual data point values
fig = px.scatter(
    df,
    x='age', y='salary',
    color='dept',
    size='score',
    hover_data=['dept', 'score'],
    title='Salary vs Age (Interactive — hover over points)'
)
fig.show()

# Interactive histogram
fig = px.histogram(df, x='salary', color='dept', barmode='overlay',
                   title='Salary Distribution by Department')
fig.show()

# Interactive box plot
fig = px.box(df, x='dept', y='salary', color='dept',
             title='Salary by Department (Interactive)')
fig.show()
```

> **When to use plotly vs seaborn:**
> - seaborn → EDA in a notebook, static plots for reports/papers
> - plotly → interactive exploration, large datasets, dashboards, presenting to business stakeholders

## Summary — The Mental Model

```
Want to see a trend over time?         →  plt.plot()
Want to see two features relate?       →  plt.scatter() / sns.scatterplot()
Want to see one feature's shape?       →  sns.histplot(kde=True)
Want to compare groups side by side?   →  sns.boxplot()
Want all feature correlations at once? →  sns.heatmap(df.corr())
Want all feature pairs at once?        →  sns.pairplot(df)
Want category counts?                  →  sns.countplot()
Want multiple plots in one figure?     →  plt.subplots(rows, cols)
Want to show classifier performance?   →  sns.heatmap(confusion_matrix)
Want training curve diagnostics?       →  plt.plot() train + val loss/accuracy
Want interactive exploration?          →  plotly express
```
