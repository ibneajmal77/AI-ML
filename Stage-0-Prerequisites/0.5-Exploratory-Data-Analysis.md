## Key Takeaways — Read This Before Any Project

1. **EDA has a fixed order:** Shape → Types → Missing → Duplicates → Distributions → Outliers → Correlations → Target relationship. Never skip steps.
2. **`df.describe()` lies about categorical columns** — always use `df.describe(include='all')` or check categoricals separately with `value_counts()`
3. **Outlier rule:** Values beyond `Q1 - 1.5×IQR` or `Q3 + 1.5×IQR` are outliers — investigate before removing. Sometimes they're real signal, sometimes they're data errors.
4. **Check target distribution first** — a heavily imbalanced target (95% one class) changes everything about how you train and evaluate your model
5. **Correlation ≠ causation** — but high correlation between a feature and the target = strong signal worth keeping
6. **Log transform skewed features** — income, price, count data are almost always skewed. `np.log1p()` before modeling dramatically improves many models
7. **Document every finding** — EDA decisions (why you dropped a column, why you log-transformed salary) must be written down. You will forget otherwise.

## SECTION 1 — The EDA Mindset

### What EDA Actually Is ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | A structured process of asking questions about your dataset before building any model. |
| **Why** | Raw data from the real world is always dirty, biased, and surprising. A model trained on unexamined data will silently learn from errors, missing values, and skewed distributions — and produce confidently wrong predictions. |
| **When** | Every time you receive a new dataset. No exceptions. |
| **Where** | Between loading data and feature engineering — always. |
| **Who** | Every ML engineer, data scientist. It separates professionals from amateurs. |
| **How** | Follow a systematic checklist — the same 7 questions every time. |
| **How often** | Every project, every dataset, including your own generated data. |

**The 7 EDA Questions — always in this order:**

| # | Question | Why it matters |
|---|---------|---------------|
| 1 | What is the shape and structure? | How big is it? What columns exist? |
| 2 | What are the data types? | Are numeric columns stored as strings? Is the date a datetime or object? |
| 3 | What is missing? | You cannot model on NaN — you need a strategy |
| 4 | Are there duplicates? | Duplicate rows inflate confidence and distort training |
| 5 | What are the distributions? | Skewed? Normal? Bimodal? Determines preprocessing |
| 6 | Are there outliers? | Data errors or real signal? Drives cleaning decisions |
| 7 | How do features relate to the target? | Which features are actually predictive? |

## SECTION 2 — Step 1: Structure and Types

### Shape, Columns, dtypes ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | Understand the size and structure of your dataset before touching any values. |
| **Why** | Before you do anything, you need to know: how many rows, how many columns, what each column is, and whether the types make sense. A `salary` column stored as `object` (string) will silently break any numeric operation. |
| **When** | First thing. Immediately after loading. |

```python
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns

sns.set_theme(style='whitegrid', palette='muted')
np.random.seed(42)

# Load or create dataset
df = pd.read_csv('data.csv')   # in real project

# Step 1 — Structure
print("Shape:", df.shape)               # (rows, cols)
print("Columns:", df.columns.tolist())  # all column names
print("\nData Types:")
print(df.dtypes)                        # look for wrong types

# Common type issues to look for:
# - salary stored as 'object' (someone put '$90,000' in the CSV)
# - date stored as 'object' instead of 'datetime64'
# - boolean stored as 0/1 integers
# - ID columns (should be excluded from modeling — not a feature)

# Fix wrong types
df['salary'] = pd.to_numeric(df['salary'].str.replace('$','').str.replace(',',''), errors='coerce')
df['date']   = pd.to_datetime(df['date'])
```

> **Real scenario:** You receive a dataset where `age` column has values like `"25 years"` — stored as string. `df.dtypes` catches this in 1 second. Without this check, your model training crashes with a cryptic error 10 minutes later. Always check types first.

## SECTION 3 — Step 2: Missing Values

### Missing Value Analysis ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | Count, locate, and understand why values are missing in your dataset. |
| **Why** | Every ML model crashes or degrades with NaN values. You need to know: how much is missing, which columns, and why — before deciding whether to fill or drop. |
| **When** | Second step in every EDA. |
| **Where** | Informs your entire preprocessing strategy. |
| **How** | Count → visualize → decide per column. |

```python
# Count missing
missing_count = df.isnull().sum()
missing_pct   = df.isnull().mean() * 100

missing_summary = pd.DataFrame({
    'missing_count': missing_count,
    'missing_pct':   missing_pct.round(2)
}).sort_values('missing_pct', ascending=False)

print(missing_summary[missing_summary['missing_count'] > 0])

# Visualize missing pattern
fig, ax = plt.subplots(figsize=(10, 4))
missing_pct[missing_pct > 0].sort_values().plot(kind='barh', ax=ax, color='tomato')
ax.set_title('Missing Value Percentage by Column')
ax.set_xlabel('% Missing')
plt.tight_layout()
plt.show()
```

**Decision rules:**

| % Missing | Decision |
|-----------|---------|
| < 5% | Fill with median (numeric) or mode (categorical) |
| 5% – 20% | Fill with group median or use a model to predict missing values |
| 20% – 50% | Decide based on feature importance — may still be worth keeping |
| > 50% | Seriously consider dropping the column |
| Target column | Always drop rows with missing target — never fill it |

```python
# After deciding — apply strategy
df['salary'].fillna(df.groupby('dept')['salary'].transform('median'), inplace=True)
df['category'].fillna(df['category'].mode()[0], inplace=True)
df.dropna(subset=['target'], inplace=True)   # never fill target
```

> **Used in Stage 1:** The Titanic dataset has 77% missing in `Cabin` (drop it), 20% missing in `Age` (fill with median by class), and 0.2% in `Embarked` (fill with mode). EDA tells you the strategy for each column differently.

## SECTION 4 — Step 3: Duplicates

### Duplicate Row Detection ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | Identical rows appearing more than once in your dataset. |
| **Why** | Duplicate rows make your model see those samples multiple times — they get more weight and distort learning. A model with 10% duplicates is essentially memorizing those rows. |
| **When** | Third step in EDA, after missing values. |
| **When to drop:** | Almost always. Unless you have a reason to believe duplicates represent real repeated events. |

```python
# Check
total_dupes = df.duplicated().sum()
print(f"Duplicate rows: {total_dupes} ({total_dupes/len(df)*100:.1f}%)")

# See them
df[df.duplicated(keep=False)].sort_values(df.columns.tolist())

# Check duplicates on specific columns (not all columns)
# e.g., same employee_id appearing twice = duplicate even if other cols differ
df.duplicated(subset=['employee_id']).sum()

# Remove
df = df.drop_duplicates()
df = df.drop_duplicates(subset=['employee_id'], keep='first')
df = df.reset_index(drop=True)
print(f"Shape after removing duplicates: {df.shape}")
```

> **Common scenario:** You join two tables and accidentally create duplicate rows. Or your data pipeline ran twice and appended the same records. Always check.

## SECTION 5 — Step 4: Distributions

### Univariate Analysis — One Feature at a Time ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | Examine the distribution of each individual feature independently. |
| **Why** | Distribution shape determines preprocessing. Normal → z-score. Skewed → log transform. Bimodal → might need to split into two features. You cannot preprocess correctly without seeing the distribution. |
| **When** | After cleaning missing values and duplicates. |
| **Where** | Drives your entire preprocessing strategy in Stage 1. |

```python
numeric_cols = df.select_dtypes(include=[np.number]).columns.tolist()

# Distribution for all numeric columns at once
n_cols = 3
n_rows = (len(numeric_cols) + n_cols - 1) // n_cols
fig, axes = plt.subplots(n_rows, n_cols, figsize=(15, n_rows * 4))

for i, col in enumerate(numeric_cols):
    ax = axes.flatten()[i] if n_rows > 1 else axes[i]
    sns.histplot(df[col].dropna(), bins=30, kde=True, ax=ax, color=f'C{i}')
    ax.set_title(f'{col}\nskew={df[col].skew():.2f}')
    ax.set_xlabel('')

# Hide unused axes
for j in range(i + 1, n_rows * n_cols):
    axes.flatten()[j].set_visible(False)

plt.suptitle('Feature Distributions', fontsize=14)
plt.tight_layout()
plt.show()
```

### Skewness — Should You Log Transform? ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | A measure of how asymmetric a distribution is. Zero = perfectly symmetric (normal). Positive = right tail (most values are small, a few are very large). |
| **Why** | Linear models assume features are roughly normal. Highly skewed features hurt linear model performance. Log transform fixes this. |
| **When** | When `df['col'].skew() > 1` or `< -1` — consider transforming. |

```python
# Check skewness
skew_scores = df[numeric_cols].skew().sort_values(ascending=False)
print("Skewness scores:")
print(skew_scores)

# Columns with skew > 1 should be log-transformed
high_skew = skew_scores[abs(skew_scores) > 1].index.tolist()
print(f"\nHighly skewed columns: {high_skew}")

# Apply log transform — np.log1p handles zeros (log(0) = -inf, log1p(0) = 0)
for col in high_skew:
    df[f'{col}_log'] = np.log1p(df[col])

# Visualize before vs after
fig, axes = plt.subplots(1, 2, figsize=(12, 4))
if high_skew:
    col = high_skew[0]
    sns.histplot(df[col].dropna(), bins=30, kde=True, ax=axes[0], color='tomato')
    axes[0].set_title(f'{col} — BEFORE (skew={df[col].skew():.2f})')
    sns.histplot(df[f'{col}_log'].dropna(), bins=30, kde=True, ax=axes[1], color='seagreen')
    axes[1].set_title(f'{col}_log — AFTER (skew={df[f"{col}_log"].skew():.2f})')
    plt.tight_layout()
    plt.show()
```

> **Used in Stage 1:** House price datasets always have heavily skewed `price` and `sqft` columns. Log-transforming them before linear regression often improves R² by 10–20%. This is one of the highest-impact preprocessing decisions.

## SECTION 6 — Step 5: Outliers

### Outlier Detection — IQR Method ⭐⭐⭐ 🧠 Pattern, 📖 Code

| | |
|---|---|
| **What** | Data points that are far from the rest of the values in a column. |
| **Why** | Outliers can be data errors (age = 999), or real extreme values (a billionaire in a salary dataset). Either way they distort model training — especially linear models and KNN which are sensitive to scale. |
| **When** | During EDA before training. |
| **How** | IQR rule: anything beyond Q1 - 1.5×IQR or Q3 + 1.5×IQR is an outlier candidate. |
| **Decision:** | Investigate first. Drop if it's a data error. Cap (clip) if it's a real but extreme value. Keep if your model handles it. |

```python
def detect_outliers_iqr(df, col):
    Q1  = df[col].quantile(0.25)
    Q3  = df[col].quantile(0.75)
    IQR = Q3 - Q1
    lower = Q1 - 1.5 * IQR
    upper = Q3 + 1.5 * IQR
    outliers = df[(df[col] < lower) | (df[col] > upper)]
    return outliers, lower, upper

# Check each numeric column
for col in numeric_cols:
    outliers, lower, upper = detect_outliers_iqr(df, col)
    if len(outliers) > 0:
        print(f"{col}: {len(outliers)} outliers ({len(outliers)/len(df)*100:.1f}%)"
              f" | valid range: [{lower:.1f}, {upper:.1f}]")

# Visualize outliers with boxplot
fig, axes = plt.subplots(1, min(3, len(numeric_cols)), figsize=(14, 4))
for i, col in enumerate(numeric_cols[:3]):
    sns.boxplot(y=df[col].dropna(), ax=axes[i], color='steelblue')
    axes[i].set_title(f'{col}')
plt.suptitle('Outlier Detection via Box Plots', fontsize=13)
plt.tight_layout()
plt.show()

# Handle outliers — 3 options:
# 1. Drop rows (use only when clearly a data error)
df_clean = df[df['age'] < 100]

# 2. Cap (clip) — replace with boundary value (preferred for real extremes)
_, lower, upper = detect_outliers_iqr(df, 'salary')
df['salary_capped'] = df['salary'].clip(lower=lower, upper=upper)

# 3. Keep — when using tree-based models (Random Forest, XGBoost) that are outlier-robust
```

> **Used in Stage 1:** KNN (K-Nearest Neighbors) is extremely sensitive to outliers because it measures distances. A salary outlier of $10M in a dataset of $50K–$100K salaries will distort every neighbor calculation. You must cap it before KNN. Tree-based models (Random Forest, XGBoost) are outlier-robust — you don't need to cap for those.

## SECTION 7 — Step 6: Correlations

### Feature Correlation Analysis ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | Measure the linear relationship between every pair of numeric features — and between features and the target. |
| **Why** | Highly correlated features add noise without adding information. Features correlated with the target are your most predictive inputs. This drives feature selection. |
| **When** | After cleaning — before feature engineering or training. |
| **Where** | Feature selection step in every ML pipeline. |

```python
# Correlation matrix
corr = df[numeric_cols].corr()

fig, ax = plt.subplots(figsize=(10, 8))
sns.heatmap(
    corr,
    annot=True, fmt='.2f',
    cmap='coolwarm', center=0,
    square=True, linewidths=0.5,
    mask=np.triu(np.ones_like(corr, dtype=bool)),  # show lower triangle only
    ax=ax
)
ax.set_title('Feature Correlation Matrix (lower triangle)')
plt.tight_layout()
plt.show()

# Find highly correlated feature pairs (potential redundancy)
threshold = 0.85
high_corr_pairs = []
for i in range(len(corr.columns)):
    for j in range(i + 1, len(corr.columns)):
        if abs(corr.iloc[i, j]) > threshold:
            high_corr_pairs.append((corr.columns[i], corr.columns[j], corr.iloc[i, j]))

print("Highly correlated pairs (may be redundant):")
for a, b, val in high_corr_pairs:
    print(f"  {a} ↔ {b}: {val:.2f}")

# Feature-target correlations — most important for feature selection
target = 'promoted'
if target in df.columns:
    target_corr = df[numeric_cols + [target]].corr()[target].drop(target).sort_values(key=abs, ascending=False)
    print(f"\nCorrelation with target '{target}':")
    print(target_corr.round(3))
```

> **What to do with high correlation between features:** If `salary` and `annual_income` correlate at 0.97 — they're measuring the same thing. Drop one. Keeping both adds noise and can cause multicollinearity issues in linear models.

> **Used in Stage 1:** Before training a logistic regression, always check feature-target correlations. The top 5 correlated features should be your starting feature set. This alone can reduce model complexity by 50% while keeping most of the predictive power.

## SECTION 8 — Step 7: Target Analysis

### Understanding Your Target Variable ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | Deep analysis of the column your model is trying to predict. |
| **Why** | The target distribution determines everything: which metric to use, how to handle imbalance, whether to transform it. This is the most important single column in your dataset. |
| **When** | As part of EDA — analyze the target separately and carefully. |
| **Where** | Informs model selection, evaluation metrics, and training strategy. |

```python
target = 'promoted'

# Classification target — distribution
print("Target distribution:")
print(df[target].value_counts())
print("\nTarget percentage:")
print(df[target].value_counts(normalize=True).round(3) * 100)

fig, axes = plt.subplots(1, 2, figsize=(12, 4))
sns.countplot(data=df, x=target, ax=axes[0], palette='Set2')
axes[0].set_title(f'Target Distribution: {target}')

# Feature vs target — see which features separate classes
sns.boxplot(data=df, x=target, y='salary', ax=axes[1], palette='Set2')
axes[1].set_title(f'Salary vs {target}')
plt.tight_layout()
plt.show()

# For all numeric features vs target — one line each
fig, axes = plt.subplots(2, 3, figsize=(15, 8))
for i, col in enumerate(numeric_cols[:6]):
    ax = axes.flatten()[i]
    sns.boxplot(data=df, x=target, y=col, ax=ax, palette='Set2')
    ax.set_title(f'{col} vs {target}')
plt.suptitle('Feature Distributions by Target Class', fontsize=13)
plt.tight_layout()
plt.show()
```

**Imbalanced target — what to do:**

```python
class_counts = df[target].value_counts()
imbalance_ratio = class_counts.min() / class_counts.max()
print(f"Imbalance ratio: {imbalance_ratio:.2f}")

# If imbalance_ratio < 0.2 — you have a serious imbalance problem
# Solutions you'll learn in Stage 1:
# - class_weight='balanced' in sklearn models
# - SMOTE (Synthetic Minority Oversampling)
# - Use F1 / AUC-ROC instead of accuracy as your metric
```

> **Used in Stage 1:** If your fraud dataset has 0.1% fraud, a model predicting "not fraud" every time achieves 99.9% accuracy but catches zero fraud. EDA reveals this before you make that mistake.

## SECTION 9 — Categorical Column Analysis

### Analyzing Non-Numeric Columns ⭐⭐⭐ 🧠

| | |
|---|---|
| **What** | Examine string/category columns — cardinality, distribution, unexpected values. |
| **Why** | Categorical columns need different treatment than numeric. High cardinality (1000 unique cities) cannot be one-hot encoded. Rare categories create noise. Typos create fake categories. |
| **When** | During EDA — analyze all categorical columns before deciding encoding strategy. |

```python
cat_cols = df.select_dtypes(include=['object', 'category']).columns.tolist()

for col in cat_cols:
    n_unique = df[col].nunique()
    top_vals = df[col].value_counts().head(5)
    print(f"\n{col}:")
    print(f"  Unique values: {n_unique}")
    print(f"  Top 5:\n{top_vals}")
    print(f"  Missing: {df[col].isnull().sum()}")

# Visualize top categories
fig, axes = plt.subplots(1, min(3, len(cat_cols)), figsize=(14, 4))
for i, col in enumerate(cat_cols[:3]):
    top = df[col].value_counts().head(10)
    ax = axes[i] if len(cat_cols) > 1 else axes
    sns.barplot(x=top.values, y=top.index, ax=ax, palette='muted')
    ax.set_title(f'{col} — Top Categories')
    ax.set_xlabel('Count')
plt.tight_layout()
plt.show()

# Cardinality decision guide:
# 2-10 unique values  → one-hot encoding (get_dummies)
# 10-50 unique values → target encoding or frequency encoding
# 50+ unique values   → frequency encoding, embedding (neural nets), or drop
```

> **Real trap:** A `city` column with 500 unique values. One-hot encoding creates 500 new columns — most with near-zero variance. This explodes your feature space and kills model performance. EDA reveals this before you make it.

## SECTION 10 — Full EDA Template

### The Complete EDA Checklist ⭐⭐⭐ 🧠

This is the template to follow for every new dataset, every time:

```python
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns

sns.set_theme(style='whitegrid', palette='muted')

def run_eda(df, target_col=None):
    print("=" * 60)
    print("STEP 1 — STRUCTURE")
    print("=" * 60)
    print(f"Shape: {df.shape}")
    print(f"Columns: {df.columns.tolist()}")
    print(f"\nData Types:\n{df.dtypes}")

    print("\n" + "=" * 60)
    print("STEP 2 — MISSING VALUES")
    print("=" * 60)
    missing = df.isnull().mean() * 100
    missing = missing[missing > 0].sort_values(ascending=False)
    if len(missing) > 0:
        print(missing.round(2).to_string())
    else:
        print("No missing values ✓")

    print("\n" + "=" * 60)
    print("STEP 3 — DUPLICATES")
    print("=" * 60)
    dupes = df.duplicated().sum()
    print(f"Duplicate rows: {dupes} ({dupes/len(df)*100:.1f}%)")

    print("\n" + "=" * 60)
    print("STEP 4 — NUMERIC SUMMARY")
    print("=" * 60)
    print(df.describe().round(2))

    print("\n" + "=" * 60)
    print("STEP 5 — SKEWNESS")
    print("=" * 60)
    num_cols = df.select_dtypes(include=[np.number]).columns
    skew = df[num_cols].skew().sort_values(ascending=False)
    print(skew.round(2))
    high_skew = skew[abs(skew) > 1]
    if len(high_skew) > 0:
        print(f"\n⚠ Highly skewed (consider log transform): {high_skew.index.tolist()}")

    print("\n" + "=" * 60)
    print("STEP 6 — CATEGORICAL COLUMNS")
    print("=" * 60)
    cat_cols = df.select_dtypes(include=['object', 'category']).columns
    for col in cat_cols:
        print(f"\n{col}: {df[col].nunique()} unique values")
        print(df[col].value_counts().head(5))

    if target_col and target_col in df.columns:
        print("\n" + "=" * 60)
        print(f"STEP 7 — TARGET: {target_col}")
        print("=" * 60)
        print(df[target_col].value_counts())
        print(f"\nImbalance ratio: {df[target_col].value_counts().min() / df[target_col].value_counts().max():.2f}")

# Usage
# run_eda(df, target_col='promoted')
```

## SECTION 11 — Automated EDA Tools

### ydata-profiling and sweetviz ⭐⭐ 📖

| | |
|---|---|
| **What** | Libraries that generate a full EDA report in one line of code. |
| **Why** | For quick exploration of a new dataset, these save 30 minutes of manual EDA. Good for a first pass — but you still need to understand what you're seeing. |
| **When** | First pass on any large dataset. Not a replacement for manual EDA — a starting point. |


## Summary — The Mental Model

```
New dataset arrives
        ↓
1. Shape & types        →  df.shape, df.dtypes — any wrong types?
        ↓
2. Missing values       →  df.isnull().mean() — how much? which columns?
        ↓
3. Duplicates           →  df.duplicated().sum() — remove them
        ↓
4. Distributions        →  sns.histplot + df.skew() — normal or skewed?
        ↓
5. Outliers             →  sns.boxplot + IQR rule — error or real?
        ↓
6. Correlations         →  sns.heatmap(df.corr()) — redundant features?
        ↓
7. Target analysis      →  value_counts + feature vs target plots — balanced? predictive features?
        ↓
Document findings
        ↓
Clean & engineer        →  You now know exactly what to fix and why
```
